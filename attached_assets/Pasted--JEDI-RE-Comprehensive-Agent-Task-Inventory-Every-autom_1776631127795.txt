## Classification Legend

| Tag | Meaning |
|-----|---------|
| **WF** | Workflow — Inngest function, deterministic sequence of steps, may invoke agents and tools |
| **T** | Tool — a single typed function an agent calls (via `platformClient`) |
| **A** | Agent — new reasoning loop required (warning: this bar is high) |
| **C** | Code — deterministic service/cron/DB trigger, no LLM, no agent |
| **+T** | Tool that needs to be added to an existing agent's registry |

**Critical finding: you do not need new agents.** The 5-agent roster (Research, Zoning, Supply, CashFlow, Commentary) handles every task in this inventory. Everything else is either deterministic code, workflows that chain tools, or tools to add to the existing agents.

## What's NOT on This List (And Shouldn't Be)

The following got rejected as "not agent work":

- Notifications, emails, SMS dispatch → deterministic service
- Stripe webhook handling → deterministic service
- Credit balance accounting → deterministic service
- PDF/Excel rendering → deterministic service
- Geocoding → deterministic service
- Monte Carlo simulation → deterministic compute
- Florida tax math → deterministic compute
- Debt schedule generation → deterministic compute
- Correlation engine updates → streaming service
- Traffic weight retraining → scheduled ML job
- Deal archive snapshots → DB triggers
- Asset class / deal type / sub-strategy classification → deterministic detection logic

None of these involve reasoning under uncertainty with tool use. They are functions, not employees. Keep them that way.
