# DISPATCH — B2b: ACCOUNT-ORIGIN PATHS (EVERY USER GETS AN ORG)

**Makes multi-user reachable.** Three ways a user comes to exist must ALL assign org membership:
signup (exists), bridge/chat (the B1 gap — no org today), invite (new, Institutional-only). Also
replaces B2a's flagged `LIMIT 1` resolution with deterministic default-org resolution (switcher-ready
for later). This is the path that creates a user's SECOND org membership, so it's where active-org
resolution stops being theoretical.
**Model (settled):** default-org NOW (session-switcher later, clean seam); invites = email-based,
Institutional-only; everyone in ≥1 org, solo = org-of-one.
**Repo:** `Nardo758/JediRe.git` — branch `claude/b2b-account-origin`.
**Verification rule (S1-01):** proven LIVE — a new user via EACH path lands in an org; an invited user
joins an existing org and shares its pool; resolution is deterministic (not arbitrary LIMIT 1).
**Report to:** `docs/audits/B2B_ACCOUNT_ORIGIN_VERDICT.md`

## PHASE 1 — MAP THE THREE PATHS (READ-ONLY, STOP)

**1. Signup path (exists — confirm it's correct).** Trace web signup → where it creates the org +

   whatever field the new resolution will read) — if signup creates an org but DOESN'T record it as

**2. Bridge path (the B1 gap — creates NO org today).** `sessionStore.ts findOrCreateUser` writes a

   also create an org (owner = the bridge user), an org_members row, and set default_org_id — same

   signup.

**3. Invite path (NEW — doesn't exist).** Design it:
   - **Where default_org_id lives.** Add it (users.default_org_id, nullable UUID, FK to
     organizations) OR confirm an existing field. This is what resolution reads.
   - **Invite mechanism (email-based):** an org OWNER invites by email → an invite record is created
     (org_id, email, token, status, expires) → invitee receives a link → accepting creates an
     org_members row (role=member) in the INVITER's org. Design the invite table + flow.
   - **Institutional-only gate:** only orgs on the Institutional tier can invite. Where is that

   sets session-active-org, so everyone resolves to default_org_id — deterministic, correct for

## PHASE 2 — BUILD (after design approved)

**Headline from Phase 1: web signup is a 404 — `/register` doesn't exist. Path 1 is building the
primary account-creation path for the first time, not modifying it. This is launch-critical: nobody
can become a customer via the web today.**

### DECISIONS (settled — build to these)
- **Signup org name:** `{firstName}'s Organization`, fall back to email-prefix if no name. NOT
  "Personal" (these become teams for Institutional).
- **Invite sender display name:** inviter's `full_name` (JOIN), fall back to email.
- **Bridge org name:** generic (`Personal Organization`) — no name available at bridge creation, and
  platform detail (WhatsApp/SMS) shouldn't surface in the workspace name.
- **Accept-invite page:** see step 5 — CONFIRM it exists; if not, building it is IN scope here.

### STEP 0 — Accept-page existence check (do first, it sizes the dispatch)
Confirm whether a frontend route/page handles the invite `accept_url` (`/accept-invite?token=...`).
- If it EXISTS → invite path is backend-only (email + gate), as scoped.
- If it does NOT exist → building it is part of THIS dispatch (a page that reads the token, calls
  `POST /invitations/:token/accept`, handles logged-in vs. needs-signup, shows success/failure). An
  invite email linking to a 404 is the same "mechanism without entry point" gap as signup being 404.
  Report which, and if building, note the added scope.

### Build steps
1. **`default_org_id` column + backfill.** `ALTER TABLE users ADD COLUMN default_org_id UUID NULL
   REFERENCES organizations(id)`. Backfill: each user's default = their single org (post-B1,
   unambiguous — Leon→dd201183; bridge/other users NULL until their next creation event sets it).
   `file:line`.

2. **Resolution replacement (orgCreditService.ts:27).** Replace `LIMIT 1` with:
   `session_active_org_id ?? users.default_org_id`, with an `org_members LIMIT 1` fallback ONLY for
   users who slip through pre-backfill. **The fallback MUST log a warning when it fires** — a real
   user hitting the LIMIT 1 fallback in production means their default_org_id didn't get set (a
   creation-path bug); it must be visible, not silently resolved. Silent fallback = the arbitrary-
   resolution bug reborn, rarer and harder to spot. `file:line`.

3. **Path 1 — BUILD signup (`POST /api/v1/auth/register`).** Net-new endpoint in
   inline-auth.routes.ts. Single transaction: INSERT users → INSERT organizations
   (`{firstName}'s Organization`) → INSERT org_members (owner) → INSERT org_credit_balances (scout
   tier, allotment-seeded) → UPDATE users SET default_org_id. All-or-nothing. `file:line`.

4. **Path 2 — Bridge org creation.** Wrap `sessionStore.ts findOrCreateUser` in a transaction: after
   the users INSERT, also create org (`Personal Organization`) + org_members (owner) +
   org_credit_balances (scout) + set default_org_id. Same shape as signup. `file:line`. (Closes B1
   gap.)

5. **Path 3 — Finish invites.** The token flow exists (org.routes.ts:117 create, :182 accept). Two
   gaps + the page:
   - **Email:** wire `emailService.sendOrgInvitation()` + template (reuse the password-reset /
     share-invitation pattern, Resend on verified jedire.com). The route currently returns
     accept_url but sends nothing. `file:line`.
   - **Institutional gate:** in POST /:orgId/invitations, after the existing access check, query
     `org_credit_balances.subscription_tier` for the org → 403 if not 'institutional'. `file:line`.
   - **Accept page:** per Step 0 — build if missing.

6. **Institutional-only enforcement** proven: non-Institutional org invite attempt → 403.

### Every creation path seeds org_credit_balances
Signup, bridge, and (implicitly) the org an invite joins all need an org_credit_balances row — B2a
created the pool concept; every new org here must get one (scout tier default for signup/bridge; the
invited org already has one). Confirm no path creates an org WITHOUT a pool (an org with no pool =
gate resolves to nothing = broken).
