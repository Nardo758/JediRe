# Console — Repurpose Selected Tab as DEAL TERMS

## What this is

You're repurposing the tab Leon has selected with the Replit selection tool.
The tab gets renamed and its content replaced. The output is a new sub-tab
inside the Console page (the F9 page formerly known as Control Center).

The Console already contains four working sub-tabs: STANCE, INPUTS (formerly
Assumptions), UNIT MIX, TAX. The selected tab will become a fifth sub-tab
called **DEAL TERMS** that consolidates four operator-decision input groups
that today are scattered across F9: Acquisition, Hold & Targets, Exit, and
(optionally) Capital Structure.

## Scope discipline — read this twice

This task is **STRUCTURE ONLY**. Build the visual scaffold, the row
structure, the input fields, and the section layouts. Do **NOT** wire any
of the inputs to backend services, the dealStore, the LayeredValue store,
or any API endpoint. Claude Code will do data wiring in a follow-up task
once the structure is approved.

That means specifically:
- Use placeholder values in every cell (e.g. `--`, `$0`, "Not Provided")
- No `useEffect` calls that fetch
- No `apiClient` imports
- No `dealStore` reads or writes
- No event emissions
- Local component `useState` is fine for the dropdowns/inputs to work
  visually, but those values reset on unmount — that's correct
- All `LayeredValue` columns render as static badges, not live data

If the four existing tabs disagree on any of these, ask Leon — do not pick

## Section structure

The tab has three required sections plus one optional. Render them
top-to-bottom with the existing Console section-header pattern. Each row
uses the same column structure as the INPUTS sub-tab.

### Column structure (every row, every section)

```
| Line Item | Broker | Platform | User Override | Resolved | Source | Derived | Flag |
```

- **Line Item** — left-aligned label, sticky-left
- **Broker** — LV layer 1 (from OM extraction). Null = `--`
- **Platform** — LV layer 2 (from agent / computed). Null = `--`
- **User Override** — LV layer 3 (editable). Empty = passthrough.
  Teal-400 border on the input
- **Resolved** — current effective value (the one the model uses).
  Highlighted column
- **Source** — badge ('Override', 'Platform', 'OM Narrative', 'Computed',
  'Not Provided' — match the badge set in INPUTS)
- **Derived** — for rows where a per-unit / per-SF / pct-of-X derived
  value is meaningful (e.g. Purchase Price → $/unit). Else blank
- **Flag** — collision indicator (broker vs platform divergence >threshold)
  or integrity warning. Else blank

For pure-operator rows (e.g. Hold Period), Broker and Platform render `--`
or `N/A` styling (slightly muted), Override is the only meaningful column,
Source shows 'Override'.
