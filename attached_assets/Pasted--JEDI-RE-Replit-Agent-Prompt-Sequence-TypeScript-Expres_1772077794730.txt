## PROMPT 4: Comp Query Engine

```
Build the comp search engine. This finds comparable properties from the user's portfolio (owned assets with uploaded actuals) and ranks them by match quality.

First, create a Drizzle query (or raw SQL if needed for the subquery) that acts as the comp pool:

The comp query should:
- Join properties → deals (where dealStatus = 'owned') → dealMonthlyActuals (last 12 months)
- Aggregate monthly actuals into trailing-12 averages: avgExpensePerUnit, avgNoiPerUnit, avgOccupancy, avgRentAchieved, avgRentPsf, monthsOfData, latestPeriod
- Only include where monthsOfData >= 3

Create a service class CompService with method:

async searchComps(params: {
  userId: string;
  propertyType: string;
  stories?: number;
  yearBuilt?: number;
  totalUnits?: number;
  latitude: number;
  longitude: number;
  maxDistanceMiles?: number; // default 10
  limit?: number; // default 10
}): Promise<CompResult[]>

The method should:
1. Query the comp pool for the given userId
2. Filter by propertyType (exact or similar — garden_style can match garden_style and mid_rise)
3. Filter by distance: use the Haversine formula in SQL or JS to calculate distance between two lat/lng pairs in miles. If PostGIS is available, use ST_DWithin. If not, use this formula in a WHERE clause:
   distance_miles = 3959 * acos(cos(radians(lat1)) * cos(radians(lat2)) * cos(radians(lng2) - radians(lng1)) + sin(radians(lat1)) * sin(radians(lat2)))
4. Calculate compScore (0-100) for each result:
   - Type match: 40 pts if propertyType AND stories match, 30 if just propertyType, 10 otherwise
   - Proximity: max(0, 25 - (distanceMiles * 5))
   - Vintage: 15 pts if yearBuilt within ±2 years, 10 if ±5, 5 otherwise
   - Scale: 10 pts if totalUnits within ±30%, 5 otherwise
   - Recency: 10 pts if latestPeriod within 3 months, 7 if within 6, 3 otherwise
5. Sort by compScore DESC, limit results

Return type CompResult:
{
  propertyId: string;
  propertyName: string; // from properties.address or a name field
  propertyType: string;
  stories: number;
  yearBuilt: number;
  totalUnits: number;
  distanceMiles: number;
  compScore: number;
  avgExpensePerUnit: number;
  avgNoiPerUnit: number;
  avgOccupancy: number;
  avgRentAchieved: number;
  monthsOfData: number;
  source: 'your_portfolio';
}

Express route:
POST /api/v1/comps/search — accepts search params, returns { comps: CompResult[], totalFound: number }
```

```

    private db: DrizzleDB,

  }): Promise<ThreeLayerProforma>
}

The generate method should:

1. Load the deal + property (need propertyType, stories, yearBuilt, totalUnits, lat/lng, submarketId)

2. MARKET DATA (Layer 2 — Platform Intelligence):
   - Get latest marketSnapshot for property's submarket
   - Extract: avgRentPsf, vacancyRate, rentGrowthYoy
   - Build market assumptions object

3. COMP DATA (Layer 3 — Your Data):
   - Call compService.searchComps() with the property's attributes
   - If comps found, calculate weighted averages (weight by compScore):
     weightedAvg = sum(comp.value * comp.compScore) / sum(comp.compScores)
   - Extract: avgExpensePerUnit, avgOccupancy, avgRentAchieved, avgRentPsf

4. TEMPLATE (Layer 1 — Template/Broker):
   - If templateId provided, load the template's assumptions

5. Build a 3-layer result for EACH assumption field:

interface ThreeLayerAssumption {
  market: number | null;      // from marketSnapshots
  comp: number | null;        // from user's owned assets
  template: number | null;    // from saved template
  recommended: number;        // weighted blend
  confidence: 'high' | 'medium' | 'low';
  sourceDetails: {
    market?: string;          // "Frisco/Prosper submarket avg, Jan 2026"
    comp?: string;            // "Weighted avg of 3 assets (Parkway: 92, Cedar: 74)"

  };

- If comp exists: 50% comp + 30% market + 20% template (or 60% comp + 40% market if no template)
- If no comp but market exists: 70% market + 30% template (or 100% market if no template)
- If only template: 100% template
- Confidence: 'high' if comp data, 'medium' if market only, 'low' if template only
```
