### A9-Finding-1 (tier divergence) — CLOSED, launch-blocking

writers** — the Stripe path (`webhookHandlers.ts` → `creditService`) writes
`user_credit_balances` exclusively, using canonical vocabulary (scout/operator/principal/institutional).
The legacy enum on `users` (free/professional/team/enterprise) is a fossil from a prior system.

**Launch-blocking consequence:** new Stripe users get `users.subscription_tier = NULL`
(registration INSERTs omit it). The four inngest allow-lists gate on this column →
NULL matches nothing → **agents (Research/Supply/Zoning/Commentary) refuse to run for every
net-new paying customer**, despite a correct `user_credit_balances` balance. The chat surface
IS the agent path, so this breaks the product for the exact cohort a launch produces.

**Invisible in dev:** the founder's dev account carries fossil `'professional'`, which happens
to match the allow-lists — so agents work in-house and silently fail for new customers.
(S1-01 trap in entitlement form.)

**Fix (corrected from COALESCE to column drop):**
1. Drop `users.subscription_tier` (a resolver over a zero-writer column is ceremony, not a fix).
2. Repoint all `users`-table tier readers at `user_credit_balances`, ideally through one shared
   read with a **deny-to-`scout` default** (unknown tier → lowest privilege, never permissive).
3. Replace the 4 inngest allow-lists (`commentary/supply/zoning/research.inngest.ts`) — they
   currently fail-open, accepting `professional` as entitled.
4. Registration INSERTs must stop creating desynced users.

**Four zombie-tier surfaces to clean in the same pass:**
