One discipline note worth surfacing:

The 7-day read-only phase before column drops is MORE important 
under Path B, not less. Under Path A, a missed reader keeps reading 
the old table — degraded but functional. Under Path B, a missed 
reader breaks at runtime when it hits the dropped column. Cost of 
miss is higher; verification before column drop is correspondingly 
critical.

Recommend the column-drop sequence be: revoke write permissions on 
target column → monitor 7 days for permission errors that would 
indicate a missed writer → revoke read permissions → monitor 7 days 
for runtime errors that would indicate a missed reader → drop column. 
Two 7-day windows rather than one, because Path B's failure mode at 
this step is more abrupt.

Otherwise proceed with the phase plan as written, substituting in-place 
column work for parallel-table work throughout.
