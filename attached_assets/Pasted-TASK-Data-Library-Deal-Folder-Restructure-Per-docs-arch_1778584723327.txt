TASK — Data Library Deal Folder Restructure

Per docs/architecture/HISTORICAL_OBSERVATIONS_SPEC.md §7.1
and Leon's request: reformat the Data Library so each deal
is its own folder with its supporting files within, including
archived deals with a status badge.

CONTEXT

Today the Data Library renders files as a flat list filtered
by attributes (city, zip_code, property_type, unit_count) via
data_library_files. There's no concept of "this file belongs
to this deal" at the schema level — the table has no deal_id
foreign key. Operators see a flat searchable inventory rather
than a deal-organized folder structure.

Leon wants per-deal folders for both active Portfolio deals
and archived deals, distinguished by a status badge derived
from deals.status. A deal becomes archived when it's removed
from the Pipeline (existing flow at backend/src/deals/
deals.service.ts already sets status='archived' and
archived_at=NOW()).

This work establishes the deal_id foreign key on
data_library_files. The same column is consumed by the
corpus-build work (separate ticket, sequenced after this one)
to set is_subject_property correctly on historical_observations
rows.

SCOPE

In scope:

  - Backfill: existing data_library_files rows where deal_id

  - Frontend: new DealFolderView component:
      Two-section layout: "Active Portfolio" on top, "Archived

      Within each section, list deals with expandable file
        accordions
      Per-deal: flat list of all files (no sub-folder by doc

  - Existing flat-attribute-filtered view kept as a separate

  - The Unaffiliated pseudo-folder shows files with deal_id IS

Out of scope:

SCHEMA MIGRATION

backend/src/db/migrations/YYYYMMDD_data_library_deal_folders.sql:

ALTER TABLE data_library_files
  ADD COLUMN IF NOT EXISTS deal_id UUID
    REFERENCES deals(id) ON DELETE SET NULL,
  ADD COLUMN IF NOT EXISTS redistribution_restricted BOOLEAN
    NOT NULL DEFAULT FALSE;

CREATE INDEX IF NOT EXISTS idx_data_library_files_deal
  ON data_library_files(deal_id)
  WHERE deal_id IS NOT NULL;

CREATE INDEX IF NOT EXISTS idx_data_library_files_restricted
  ON data_library_files(redistribution_restricted)
  WHERE redistribution_restricted = TRUE;

COMMENT ON COLUMN data_library_files.deal_id IS
  'Optional foreign key to deals.id. NULL for files not
   associated with a specific deal (e.g., email archive
   imports, market-wide data uploads).';

COMMENT ON COLUMN data_library_files.redistribution_restricted IS
  'TRUE for files containing licensed data that cannot be
   redistributed (e.g., CoStar, Yardi Matrix). Multi-tenant
   surfaces must filter these rows out.';

BACKFILL STRATEGY

The migration adds nullable deal_id. Existing rows stay NULL by
default (Unaffiliated). A separate one-time backfill script in
backend/src/scripts/backfill-data-library-deal-ids.ts attempts
to associate existing files where possible:

1. Files uploaded via a deal-specific upload route should have
   deal context in their original request — check audit logs
   or related tables for the deal association.
2. Files mentioned in deal_documents or deal_document_files
   tables can be joined back to their deal.
3. Files whose file_path includes a deal identifier in the
   storage hierarchy can be parsed for the deal_id.
4. Files that can't be associated stay NULL (legitimate
   Unaffiliated cases).

Report after backfill: N files associated, M files left as
Unaffiliated. The report is informational; no failure if
many files remain Unaffiliated.
