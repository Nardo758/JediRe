# M33 — Corporate Health Intelligence Engine
## The Meso-Level Leading Indicator Nobody Tracks

> **Module Decision:** "Is this submarket's demand base strengthening or weakening — and does the market know yet?"

---

## 1. THESIS

Real estate analysis has two blind spots:

1. **Macro indicators (M28)** are too broad — Fed rate changes, GDP, unemployment move entire metros. They don't tell you that Westchase (Tampa) is about to lose 800 tech jobs while Brandon (Tampa) is gaining 1,200 healthcare jobs.

2. **Deal-level metrics (M05/M07)** are too lagging — by the time vacancy rises, rent growth stalls, or absorption slows, the corporate decision that caused it happened 6–18 months ago.

**M33 sits in the middle.** Publicly traded companies report earnings, headcount, facility plans, and guidance quarterly — creating a real-time signal layer about submarket-specific demand that RE professionals never systematically track. The stock price moves first, the earnings call confirms it, the lease decision follows, and the RE metrics catch up last.

**The arbitrage:** Buy into submarkets where corporate health is improving before RE metrics reflect it. Sell out of submarkets where corporate health is deteriorating before RE metrics punish you.

## 2. CORE CONCEPTS

### 2.1 Employer Concentration Mapping

Every submarket has an employer DNA — a fingerprint of which companies, industries, and employment types drive demand. M33 maps this systematically:

```
EMPLOYER_DNA(submarket) = {
  employers: [
    {
      company: "Chewy",
      ticker: "CHWY",
      is_public: true,
      hq_or_office: "office",
      estimated_local_employees: 800,
      naics: "454110",           // Electronic Shopping
      facility_type: "corporate_hq",
      lease_expiry_known: false,
      submarket_employment_share: 0.034,  // 3.4% of submarket jobs
    },
    // ... all major employers (>50 local employees)
  ],
  concentration_metrics: {
    herfindahl_index: 0.042,           // <0.05 = diversified, >0.15 = concentrated
    top_5_share: 0.18,                 // Top 5 employers = 18% of jobs
    public_company_coverage: 0.41,     // 41% of jobs traceable to public companies
    single_employer_max_share: 0.065,  // No single employer >6.5%
    industry_diversity_score: 78,      // 0-100, higher = more diverse
  }
}
```

**Why this matters:** A submarket where one employer = 25% of demand is fundamentally riskier than one where the top 5 employers combined = 15%. M33 quantifies this, and — critically — tracks it forward using corporate earnings data.

### 2.2 Corporate Health Score (CHS)

For each public company with a meaningful local presence, M33 computes a Corporate Health Score:

```
CHS(company, quarter) = (
  Revenue_Momentum    × 0.25 +
  Earnings_Trajectory × 0.20 +
  Headcount_Signal    × 0.25 +
  Guidance_Sentiment  × 0.15 +
  Stock_Momentum      × 0.15
)
Scale: 0–100 where 70+ = Healthy, 40–69 = Watch, <40 = Stress
```

**Component Definitions:**

| Component | Formula | Data Source | Update Freq |
|-----------|---------|-------------|-------------|
| Revenue Momentum | `(rev_ttm / rev_ttm_4q_ago - 1) × 100`, normalized to 0–100 | SEC EDGAR / Yahoo Finance API | Quarterly |
| Earnings Trajectory | `EPS beat/miss streak + margin trend (3Q)`, normalized | SEC EDGAR / Yahoo Finance API | Quarterly |
| Headcount Signal | `(current_employees / employees_4q_ago - 1) × 100`, normalized | 10-K/10-Q filings, LinkedIn (proxy) | Quarterly |
| Guidance Sentiment | NLP score from earnings call transcript: expansion/contraction language | Earnings call transcripts (SEC / free APIs) | Quarterly |
| Stock Momentum | `(price_90d / price_180d - 1) × 100`, normalized to 0–100 | Yahoo Finance API (free) | Daily |

### 2.3 Submarket Corporate Health Index (SCHI)

The CHS scores roll up to a submarket-level composite:

```
SCHI(submarket) = Σ(CHS_i × employment_share_i × public_weight_adjustment)

where:
  CHS_i = Corporate Health Score for employer i
  employment_share_i = employer i's share of submarket employment
  public_weight_adjustment = 1 / public_company_coverage
    (scales up public company signals to represent full employment base)
```

**Interpretation:**
- SCHI > 70: Corporate demand base strengthening — leading indicator of future rent growth, absorption
- SCHI 50–70: Stable — no divergence signal
- SCHI < 50: Corporate demand base weakening — leading indicator of future vacancy, rent pressure
- SCHI trending (±10pts over 2 quarters): Stronger signal than absolute level

### 2.4 Corporate-RE Divergence Score (The Alpha Signal)

This is the money metric. It compares what corporate data says about a submarket's future vs. what RE metrics say about its present:

```
DIVERGENCE(submarket) = SCHI(submarket) - RE_HEALTH(submarket)

where RE_HEALTH = normalized composite of:
  - Current vacancy rate (inverted, 0-100)
  - Trailing 12-month rent growth (normalized, 0-100)
  - Trailing 12-month absorption (normalized, 0-100)
  - Cap rate trend (compression = positive, normalized, 0-100)

DIVERGENCE > +15: BULLISH DIVERGENCE
  → Corporate health says demand is coming, RE metrics haven't reflected it yet
  → ACTION: Buy before the market catches up

DIVERGENCE < -15: BEARISH DIVERGENCE
  → Corporate health says demand is weakening, RE metrics haven't reflected it yet
  → ACTION: Sell/avoid before the market catches up

|DIVERGENCE| < 15: NO SIGNAL
  → Corporate and RE metrics are aligned — no timing advantage
```

-- Master employer registry per submarket
CREATE TABLE submarket_employers (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  submarket_id          UUID NOT NULL REFERENCES submarkets(id),
  company_name          TEXT NOT NULL,
  ticker                TEXT,                          -- NULL for private companies
  is_public             BOOLEAN NOT NULL DEFAULT false,
  naics_code            TEXT,

    'corporate_hq', 'regional_office', 'distribution_center',

  )),
  estimated_local_employees  INTEGER,
  employment_share           DECIMAL(5,4),             -- 0.0000–1.0000
  data_source                TEXT,                     -- 'qcew', 'manual', 'sec_10k', 'linkedin'
  confidence                 DECIMAL(3,2),             -- 0.00–1.00
  last_verified_at           TIMESTAMPTZ,
  created_at                 TIMESTAMPTZ DEFAULT NOW(),
  updated_at                 TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(submarket_id, company_name, facility_type)
);

-- Corporate financial snapshots (quarterly)
CREATE TABLE corporate_financials (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ticker                TEXT NOT NULL,
  fiscal_quarter        TEXT NOT NULL,                 -- '2026-Q1'
  fiscal_year           INTEGER NOT NULL,
  revenue_ttm           BIGINT,                        -- cents
  revenue_yoy_pct       DECIMAL(8,4),
  eps_actual            DECIMAL(10,4),
  eps_estimate          DECIMAL(10,4),
  eps_surprise_pct      DECIMAL(8,4),
  net_income_ttm        BIGINT,
  operating_margin      DECIMAL(8,4),
  total_employees       INTEGER,
  employee_yoy_pct      DECIMAL(8,4),
  capex_ttm             BIGINT,
  free_cash_flow_ttm    BIGINT,
  guidance_sentiment    DECIMAL(5,2),                  -- -100 to +100 from NLP
  guidance_raw_text     TEXT,                           -- Key quotes from earnings call
  data_source           TEXT DEFAULT 'sec_edgar',
  filed_at              TIMESTAMPTZ,
  created_at            TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(ticker, fiscal_quarter)
);

-- Daily stock price cache
CREATE TABLE corporate_stock_prices (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ticker                TEXT NOT NULL,
  price_date            DATE NOT NULL,
  close_price           DECIMAL(12,4),
  volume                BIGINT,
  market_cap            BIGINT,
  price_90d_ago         DECIMAL(12,4),                 -- cached for momentum calc
  price_180d_ago        DECIMAL(12,4),
  stock_momentum_score  DECIMAL(5,2),                  -- pre-computed 0–100
  created_at            TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(ticker, price_date)
);

-- Computed: Corporate Health Score per company per quarter
CREATE TABLE corporate_health_scores (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ticker                TEXT NOT NULL,
  quarter               TEXT NOT NULL,                 -- '2026-Q1'
  revenue_momentum      DECIMAL(5,2),                  -- 0–100
  earnings_trajectory   DECIMAL(5,2),
  headcount_signal      DECIMAL(5,2),
  guidance_sentiment    DECIMAL(5,2),
  stock_momentum        DECIMAL(5,2),
  composite_chs         DECIMAL(5,2),                  -- Weighted composite
  chs_delta_qoq         DECIMAL(6,2),                  -- Change from prior quarter
  health_tier           TEXT CHECK (health_tier IN ('healthy', 'watch', 'stress')),
  computed_at           TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(ticker, quarter)
);

-- Computed: Submarket Corporate Health Index
CREATE TABLE submarket_corporate_health (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  submarket_id          UUID NOT NULL REFERENCES submarkets(id),
  quarter               TEXT NOT NULL,
  schi_score            DECIMAL(5,2),                  -- 0–100
  schi_delta_qoq        DECIMAL(6,2),
  re_health_score       DECIMAL(5,2),                  -- RE metrics composite
  divergence_score      DECIMAL(6,2),                  -- SCHI - RE_HEALTH
  divergence_signal     TEXT CHECK (divergence_signal IN (
    'bullish_divergence', 'bearish_divergence', 'aligned'
  )),
  herfindahl_index      DECIMAL(6,4),
  top_5_share           DECIMAL(5,4),
  public_coverage       DECIMAL(5,4),
  employer_count        INTEGER,
  public_employer_count INTEGER,
  sector_breakdown      JSONB,                         -- {naics: share} map
  computed_at           TIMESTAMPTZ DEFAULT NOW(),

  UNIQUE(submarket_id, quarter)
);

-- Facility expansion/contraction events
CREATE TABLE corporate_facility_events (
  id                    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  ticker                TEXT,
  company_name          TEXT NOT NULL,
  event_type            TEXT CHECK (event_type IN (
    'expansion', 'contraction', 'relocation_in', 'relocation_out',
    'new_lease', 'lease_expiry', 'layoff', 'hiring_announcement'
  )),
  submarket_id          UUID REFERENCES submarkets(id),
  estimated_job_impact  INTEGER,                       -- positive = jobs added
  estimated_sf_impact   INTEGER,                       -- positive = SF added
  announced_at          DATE,
  effective_at          DATE,
  source_url            TEXT,
  source_type           TEXT,                          -- 'warn_act', '10k', 'news', 'press_release'

  created_at            TIMESTAMPTZ DEFAULT NOW()

-- Indexes

CREATE INDEX idx_employers_ticker ON submarket_employers(ticker);

### 6.2 Key Service Pattern: `corporate-health-score.service.ts`

```typescript
import { db } from '../../db';
import { corporateFinancials, corporateStockPrices, corporateHealthScores } from '../../db/schema';

interface CorporateHealthInput {
  ticker: string;
  quarter: string;
}

interface CorporateHealthResult {
  ticker: string;
  quarter: string;
  components: {
    revenueMomentum: number;
    earningsTrajectory: number;
    headcountSignal: number;
    guidanceSentiment: number;
    stockMomentum: number;
  };
  compositeCHS: number;
  healthTier: 'healthy' | 'watch' | 'stress';
  deltaQoQ: number;
}

const WEIGHTS = {
  revenueMomentum: 0.25,
  earningsTrajectory: 0.20,
  headcountSignal: 0.25,
  guidanceSentiment: 0.15,
  stockMomentum: 0.15,
};

export async function computeCorporateHealthScore(
  input: CorporateHealthInput
): Promise<CorporateHealthResult> {
  // 1. Pull financial data for this quarter + prior 4 quarters
  const financials = await db.select()
    .from(corporateFinancials)
    .where(eq(corporateFinancials.ticker, input.ticker))
    .orderBy(desc(corporateFinancials.fiscal_quarter))
    .limit(5);

  // 2. Pull stock momentum from latest price record
  const latestPrice = await db.select()
    .from(corporateStockPrices)
    .where(eq(corporateStockPrices.ticker, input.ticker))
    .orderBy(desc(corporateStockPrices.price_date))
    .limit(1);

  // 3. Compute each component (0–100 normalized)
  const revenueMomentum = normalizeRevenueMomentum(financials);
  const earningsTrajectory = normalizeEarningsTrajectory(financials);
  const headcountSignal = normalizeHeadcountSignal(financials);
  const guidanceSentiment = financials[0]?.guidance_sentiment
    ? normalizeGuidance(financials[0].guidance_sentiment)
    : 50; // neutral if no transcript
  const stockMomentum = latestPrice[0]?.stock_momentum_score ?? 50;

  // 4. Weighted composite
  const compositeCHS =
    revenueMomentum * WEIGHTS.revenueMomentum +
    earningsTrajectory * WEIGHTS.earningsTrajectory +
    headcountSignal * WEIGHTS.headcountSignal +
    guidanceSentiment * WEIGHTS.guidanceSentiment +
    stockMomentum * WEIGHTS.stockMomentum;

  // 5. Tier classification
  const healthTier = compositeCHS >= 70 ? 'healthy'
    : compositeCHS >= 40 ? 'watch'
    : 'stress';

  // 6. QoQ delta
  const priorCHS = await getPriorQuarterCHS(input.ticker, input.quarter);
  const deltaQoQ = priorCHS ? compositeCHS - priorCHS : 0;

  // 7. Persist
  await db.insert(corporateHealthScores).values({
    ticker: input.ticker,
    quarter: input.quarter,
    revenue_momentum: revenueMomentum,
    earnings_trajectory: earningsTrajectory,
    headcount_signal: headcountSignal,
    guidance_sentiment: guidanceSentiment,
    stock_momentum: stockMomentum,
    composite_chs: compositeCHS,
    chs_delta_qoq: deltaQoQ,
    health_tier: healthTier,
  }).onConflictDoUpdate({
    target: [corporateHealthScores.ticker, corporateHealthScores.quarter],
    set: { composite_chs: compositeCHS, health_tier: healthTier, computed_at: new Date() },
  });

  return {
    ticker: input.ticker,
    quarter: input.quarter,
    components: { revenueMomentum, earningsTrajectory, headcountSignal, guidanceSentiment, stockMomentum },
    compositeCHS,
    healthTier,
    deltaQoQ,
  };
}
```

### 6.3 Earnings Transcript NLP (Claude as Compute Engine)

Following the established JEDI RE pattern, Claude processes earnings call transcripts to extract guidance sentiment:

```typescript
// earnings-transcript-nlp.service.ts
import Anthropic from '@anthropic-ai/sdk';

const GUIDANCE_PROMPT = `You are analyzing an earnings call transcript for real estate demand signals.
Extract ONLY information relevant to:
1. Headcount plans (hiring, layoffs, freeze)
2. Facility/office plans (expansion, consolidation, relocation)
3. Geographic market commentary (growth markets, exit markets)
4. Forward guidance tone (confident expansion vs cautious contraction)

Respond ONLY in JSON:
{
  "sentiment_score": <number -100 to +100>,
  "headcount_direction": "expanding" | "stable" | "contracting",
  "facility_plans": [{ "type": "expansion|contraction|relocation", "location_hint": string, "magnitude": string }],
  "key_quotes": [<max 3 relevant quotes, under 15 words each>],
  "confidence": <0.0 to 1.0>
}`;

export async function analyzeEarningsTranscript(
  transcript: string,
  ticker: string
): Promise<GuidanceSentimentResult> {
  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 1000,
    system: GUIDANCE_PROMPT,
    messages: [{ role: 'user', content: `Ticker: ${ticker}\n\nTranscript excerpt:\n${transcript.slice(0, 8000)}` }],
  });

  const parsed = JSON.parse(response.content[0].text);
  return parsed;
}
```

## 8. CROSS-MODULE INTEGRATION MAP

M33 is a **feeder module** — it produces intelligence consumed by many other modules but doesn't depend on their output (except M05 for RE_HEALTH baseline).

### 8.1 M33 → Other Modules

| Target Module | What M33 Feeds | How It's Used | Formula |
|---------------|---------------|---------------|---------|
| **M25 JEDI Score** | F68 Divergence Score | Modifies Demand signal weight: bullish divergence → boost Demand by up to +8pts, bearish → penalize by up to -8pts | `demand_adjusted = demand_base + (divergence / 15 × 8)` capped at ±8 |
| **M06 Demand Intelligence** | F67 SCHI + F71 Facility Events | Corporate health becomes a demand driver alongside population, jobs, infrastructure. Facility events generate demand event entries. | Facility expansion = new demand event with `impact = estimated_jobs × avg_salary / market_avg_rent` |
| **M08 Strategy Arbitrage** | F68 Divergence + F66 Concentration | Bullish divergence boosts BTS/Rental confidence. High concentration + stress = penalize long-hold strategies. | BTS demand_confidence = `base × (1 + divergence/100)` |
| **M14 Risk Assessment** | F72 Corporate Risk Exposure | New 7th risk sub-component: "Corporate Concentration Risk." High Herfindahl + any employer in 'stress' tier = elevated risk. | `corp_risk = herfindahl_normalized × (1 - min_chs/100) × 100` |
| **M20 Exit Strategy** | F68 Bearish Divergence | Bearish divergence in owned asset's submarket triggers exit timing alert. Corporate health declining = sell signal. | RSS gets new sub-component: `corporate_health_momentum × 0.10` (reduces other weights proportionally) |

## 12. VALIDATION & BACKTESTING FRAMEWORK

Before trusting M33 signals for live deals, backtest the divergence thesis:

```
BACKTEST DESIGN:
1. For each submarket in Tampa/Orlando/Miami (2019–2025):
   a. Compute historical SCHI from archived financials
   b. Compute historical RE_HEALTH from CoStar/RentCast archived data
   c. Compute DIVERGENCE at each quarter
   d. Measure: Did bullish divergence predict rent growth 2–4 quarters later?
   e. Measure: Did bearish divergence predict vacancy increase 2–4 quarters later?

2. Key test cases:
   - Spirit Airlines bankruptcy (SAVE) → Plantation/Sunrise submarket impact
   - Amazon HQ2 effect on Arlington (control: other metros without HQ2)
   - COVID-era tech layoffs → impact on Brickell/Wynwood
   - Disney hiring waves → impact on I-Drive/Kissimmee

3. Success criteria:
   - Divergence signal leads RE metric changes by ≥2 quarters in >60% of cases
   - False positive rate <30% (divergence signals that didn't materialize)
   - Magnitude correlation: larger divergence → larger eventual RE impact
```

## 14. COMPETITIVE MOAT

Nobody does this systematically for CRE. Here's why:

1. **Institutional investors** track their own portfolio company exposure but don't share it and don't map it spatially to submarkets.
2. **CBRE/JLL research** publishes annual employer reports per metro but not submarket-level, not quarterly, and not scored.
3. **CoStar** has employer data in their analytics but doesn't connect it to stock performance or earnings sentiment.
4. **Bloomberg Terminal** (the real one) tracks corporate data for equity analysts — but equity analysts don't think about which submarket Chewy's jobs sit in.

**JEDI RE's advantage:** We bridge equity market intelligence → spatial submarket mapping → RE deal scoring. The data is all free/cheap. The moat is the synthesis layer and the fact that nobody else is doing the wiring.
