Touchpoint 3 — buildSystemPrompt() classification rules (line 142-149):
- PARSER_MISS: The document clearly contains a value for this row but the extracted value is null/zero.
- PARSER_INCORRECT: The extracted value differs from the source document value by >5%.
- RANGE_ANOMALY: The extracted value is technically present but implausible (e.g. NOI/unit > $30,000/year or < $500/year for multifamily).
- INCONSISTENCY: Internal contradiction within the same document (e.g. stated NOI ≠ Revenue − Expenses).
- SEED_PLUMBING_WRITE_RACE: A value was extracted and stored in broker_claims but did not propagate to the Pro Forma year1 slot. Source-write and seed-write timestamps are within 5 minutes of each other, suggesting a pipeline write-race issue between routeOM and routeExtractionResult.
- SEED_PLUMBING_STALE_SEED: A value exists in broker_claims now, but the seed was written significantly earlier (>5 minutes before the source-write timestamp). The seeder ran when the value was not yet present; broker data was entered or extraction completed after seed creation.
- NOT_IN_DOC: The document was checked but genuinely does not contain a value for this row. Surface this as an info-level finding so the user knows the column was verified absent rather than skipped. This is positive information: confirmation that the field was looked for and not found.
- CROSS_DOC_VARIANCE: Value differs materially from another uploaded document's data for the same field.
- LOW_CONFIDENCE_EXTRACTION: Extracted value present but the document text is ambiguous or unclear.

The user prompt will indicate, per detected seed gap, the source-write timestamp and the seed-write timestamp. Use the timestamp delta to choose between SEED_PLUMBING_WRITE_RACE and SEED_PLUMBING_STALE_SEED. Where timestamps are unknown (legacy data), default to SEED_PLUMBING_STALE_SEED as the more common case.

NEVER flag EXTRACTION_OK — only surface genuine problems or verified absences (NOT_IN_DOC).
