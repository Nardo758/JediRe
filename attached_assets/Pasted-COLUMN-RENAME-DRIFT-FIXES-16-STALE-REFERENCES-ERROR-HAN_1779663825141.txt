COLUMN-RENAME DRIFT FIXES — 16 STALE REFERENCES + ERROR HANDLING

CONTEXT
Audit found 16 stale references to renamed columns across 10 files. 
The unifying mechanism: PostgreSQL 42703 undefined_column errors are 
silently swallowed by .catch(logger.warn) patterns. Most consequential 
finding: bulk upload to data_library_files has been silently broken — 
files land in storage but never get DB rows.

This dispatch fixes the 16 stale references and patches the error-
swallowing pattern at the specific sites that produced silent failures.

═══════════════════════════════════════════════════════════════════
PRINCIPLES
═══════════════════════════════════════════════════════════════════

1. CRITICAL → HIGH → MEDIUM → LOW in that order. The bulk upload 
   fix can't wait; the cosmetic fallback cleanups can.

2. EACH FIX VERIFIED INDEPENDENTLY. After fixing each Critical site, 
   prove the fix works by triggering the workflow and observing 
   the DB row that should now exist. Don't batch verification.

3. ERROR-HANDLING POLICY UPDATE AT FIX SITES. Where stale columns 
   were swallowing 42703 errors, change the .catch to surface 
   schema errors (don't swallow them). Other categories of errors 
   can remain non-fatal where the design intent was best-effort.

4. NO SCOPE CREEP. Fix only the 16 references identified in the 
   audit. Don't refactor the .catch pattern across the entire 
   codebase — only at the sites this fix touches.
