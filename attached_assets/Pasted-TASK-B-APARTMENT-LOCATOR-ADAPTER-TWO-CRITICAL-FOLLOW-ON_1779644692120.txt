CONTEXT
Tasks A + C shipped clean. property_descriptions now receives 
write-back from the orchestrator, the producer/reader contract is

via the API.

This dispatch wires the Apartment Locator scrape feed into the 
orchestrator. It also addresses two issues surfaced by the A+C 
closing note:

(1) The historical Apartment Locator ingest produced 53 duplicate 
    pairs of apartment_locator_id values. The new unique partial 
    index on (source_type, source_record_id) will now reject 
    duplicate inserts at the DB layer. The adapter MUST be 
    duplicate-safe via UPSERT or pre-check pattern, or the daily 
    scrape will start failing silently on every duplicate.

(2) The municipal_lookup step extracts assessed_value, 
    appraised_value, and owner — all valuable fields with no 
    column in property_descriptions. These are not minor: 
    assessed_value drives tax-formula reversal for Comp ProForma 
    reconstruction; appraised_value is the county's market valuation; 
    owner is critical for off-market deal sourcing. This dispatch 
    adds the columns and extends the write-back path so the data 
    is no longer log-only.

STEP 5 — APARTMENT LOCATOR SOURCE ADAPTER
═══════════════════════════════════════════════════════════════════

Location: src/services/intake-sources/apartment-locator/

A. NORMALIZE FUNCTION

   Convert a scrape record into an IntakeJob payload:
   {
     source_type: 'apartment_locator',
     source_record_id: <scrape's primary ID — confirmed in Step 1>,
     raw_input: <full original scrape record as jsonb>,
     parcel_id: <if extractable from scrape; can be null>,
     status: 'pending',
   }

B. IDEMPOTENCY VIA UPSERT

   Use INSERT ... ON CONFLICT (source_type, source_record_id) 
   WHERE source_type IS NOT NULL AND source_record_id IS NOT NULL 
   DO UPDATE SET:
   - raw_input = EXCLUDED.raw_input (refresh in case scrape updated)
   - updated_at = NOW()
   - DO NOT touch status, attempts, or enrichment_log (preserve 
     in-flight processing state)
   
   This means re-scraping the same record refreshes the raw_input 
   but doesn't restart processing. Correct semantics.

C. REPLACE CURRENT "DUMP INTO DB" PATH

   - Find the current ingest code (identified in Step 1)
   - Replace direct table writes with this adapter's normalize → 
     upsert flow
   - Preserve any non-orchestrator-relevant data writes (e.g., if 
     scrape data is also written to a separate scrape-archive table 
     for analytics, leave that alone)

D. ORCHESTRATOR INTEGRATION

   No orchestrator changes needed — it picks up status='pending' 
   jobs on the next poll cycle. Confirm by manually triggering a 
   poll after creating a test job.
