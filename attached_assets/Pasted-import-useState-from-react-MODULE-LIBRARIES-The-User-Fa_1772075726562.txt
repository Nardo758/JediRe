The comp scoring algorithm (property type match 40pts + proximity 25pts + vintage 15pts + scale 10pts + recency 10pts) runs as a SQL query against the materialized view, returning ranked comps with confidence scores."}

Every upload triggers three downstream effects: (1) comp_eligible_properties materialized view is flagged for refresh, (2) traffic_predictions validation pipeline checks for new actual-vs-predicted comparisons, (3) training_datasets table checks if retraining threshold is met (minimum 3 months data, 5+ properties per segment). The user sees their data status here; the backend sees training fuel."}

This powers M09 ProForma Layer 2 (Platform Intelligence) defaults."}

Users see which of their properties contribute to which models, accuracy metrics over time, and where data gaps exist. This transparency builds trust — users can verify that their data contributions are actually improving predictions, not just disappearing into a black box. The confidence_map feature visualizes data density across submarkets, directly correlating to prediction quality."}
