What's missing that needs to be added
1. Thesis tab (new, in Operator cluster)
This is where OperatorStance lives, plus the deal-thesis decisions that don't have a home today:

Hold period
Target unlevered/levered IRR
Capital structure preference
Exit cap stance
OperatorStance dials (rate environment, cycle position, underwriting posture, stress overlays)

Putting these on Assumptions would muddy it — Assumptions is the layered values ledger. Thesis is the operator's framing. They're sibling concepts, not the same job. Thesis answers "what's the deal thesis"; Assumptions answers "what numbers are we using and where did they come from."
2. Sources tab (new, in Audit cluster)
Document manifest, agent run history, cross-validation flags. Read-mostly. This was the "what's left over after folding provenance into rows" piece — keeping it as its own dim-gray tab keeps it discoverable for DD/lender handoff without polluting the Operator cluster.
3. Per-row provenance drawers on Assumptions (enhancement, not new tab)
Each LayeredValue row gets an expandable drawer showing the layer stack, source citations, and override entry. This is where "fold provenance in" actually lands — inside the existing Assumptions tab, not on Console.
