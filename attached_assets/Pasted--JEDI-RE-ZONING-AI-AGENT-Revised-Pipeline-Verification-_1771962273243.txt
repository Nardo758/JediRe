# JEDI RE — ZONING AI AGENT
## Revised Pipeline: Verification-First with Source Transparency

---

## THE REFRAME

The old pipeline assumed the user was asking "What's my zoning?" That's wrong.

The user already sees the zoning designation on the property boundary page. By the time they hit the Zoning & Entitlements module, they're asking two different questions:

**Question 1:** "Is this zoning designation correct, and can I see the actual code?"
**Question 2:** "Now that I trust the data, what does it mean for what I can build?"

The old 8-step pipeline tried to answer both at once and skipped the trust-building step entirely. That's backwards. If the user doesn't trust the zoning designation, nothing else matters — the capacity calculations, the entitlement paths, the strategy recommendations are all built on a foundation the user hasn't verified.

The new pipeline puts verification and source transparency FIRST, then layers intelligence on top of confirmed data.

## STEP 1: SOURCE RESOLUTION

### "Where does this zoning designation actually come from?"

The agent's first job is to FIND the authoritative source for the zoning code that the property boundary page is showing. This isn't a lookup — it's a source-tracking operation.

```
│                                                                                         │

│                                                                                         │
```

## STEP 2: VERIFY & CONFIRM

### "Is what the property boundary page says actually correct?"

This is the critical trust-building step. The agent cross-checks the GIS designation against the authoritative source and surfaces any discrepancies.

```
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│  STEP 2: VERIFY & CONFIRM                                                                │
│  ════════════════════════                                                                │
│                                                                                         │
│  INPUT:                                                                                  │
│  • Property boundary page says: "MRC-3"                                                  │
│  • Source (from Step 1): Municode §16-18A.007                                            │
│  • Parcel GIS coordinates                                                                │
│                                                                                         │
│  Agent runs FOUR verification checks:                                                    │
│                                                                                         │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐ │
│  │  CHECK 1: DESIGNATION MATCH                                                         │ │
│  │  ─────────────────────────                                                          │ │
│  │                                                                                     │ │
│  │  Does the GIS zoning field match the municipality's official zoning map?             │ │
│  │                                                                                     │ │
│  │  Method: Compare boundary page designation against:                                 │ │
│  │  a) Municipal zoning map layer (if available via ArcGIS/WMS)                        │ │
│  │  b) County GIS zoning field (the boundary page's own source)                        │ │
│  │  c) Recent rezone decisions (scraped from permit portal / council minutes)           │ │
│  │                                                                                     │ │
│  │  Outcomes:                                                                          │ │
│  │  ✅ CONFIRMED — All sources agree: MRC-3                                             │ │
│  │  ⚠️ STALE — GIS says MRC-3 but a rezone to MRC-4 was approved 45 days ago           │ │
│  │  ⚠️ SPLIT — Parcel straddles MRC-3 / MRC-2 boundary (0.8 acres in MRC-2)           │ │
│  │  ❌ CONFLICT — GIS says C-2 but municipal map shows MRC-3 (GIS data error)          │ │
│  │                                                                                     │ │
│  └─────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                         │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐ │
│  │  CHECK 2: OVERLAY DETECTION                                                         │ │
│  │  ─────────────────────────                                                          │ │
│  │                                                                                     │ │
│  │  The property boundary page typically shows BASE zoning only.                       │ │
│  │  Overlays are almost never in the GIS zoning field — this is where                  │ │
│  │  the agent adds critical value the boundary page can't.                             │ │
│  │                                                                                     │ │
│  │  Method: Spatial query against overlay district boundaries:                         │ │
│  │  a) Special Public Interest districts (SPIs)                                        │ │
│  │  b) Historic preservation overlays                                                  │ │
│  │  c) Transit-oriented development overlays                                           │ │
│  │  d) Planned development districts                                                   │ │
│  │  e) Environmental overlays (flood, wetland, watershed)                              │ │
│  │  f) Design review / corridor overlays                                               │ │
│  │                                                                                     │ │
│  │  Result for 847 Peachtree:                                                          │ │
│  │  ⚠️ OVERLAY FOUND: SPI-16 Midtown (not shown on boundary page)                     │ │
│  │     Source: Municode §16-18M.001                                                    │ │
│  │     URL: [clickable Municode link]                                                  │ │
│  │     Impact: Additional design requirements, active ground-floor mandate             │ │
│  │                                                                                     │ │
│  │  This is often the most valuable check — overlays modify base rules                 │ │
│  │  in ways the boundary page doesn't surface.                                         │ │
│  │                                                                                     │ │
│  └─────────────────────────────────────────────────────────────────────────────────────┘ │
│                                                                                         │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐ │
│  │  CHECK 3: RECENT CHANGES                                                            │ │
│  │  ────────────────────────                                                           │ │
│  │                                                                                     │ │
│  │  Has anything changed since the GIS data was last updated?                          │ │
│  │                                                                                     │ │
│  │  Method: Check JEDI's amendment tracking against code text:                         │ │
│  │  a) Municode last-updated date vs GIS last-refresh date                             │ │
│  │  b) Council meeting minutes referencing this district or parcel                     │ │
│  │  c) Pending legislation that would affect this district                             │ │
│  │                                                                                     │ │
│  │  Outcomes:                                                                          │ │
│  │  ✅ CURRENT — No amendments since last GIS refresh                                   │ │
│  │  ⚠️ AMENDED — MRC-3 parking requirements updated Nov 2025                           │ │
```
