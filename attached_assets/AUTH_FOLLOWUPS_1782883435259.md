## PART 1 — bypassAuth INVARIANT COMMENT (in-code tripwire)

`res.locals.bypassAuth` is now load-bearing across 3 sites: the floor sets it
(`index.replit.ts`), and `requireAuth` + `requireSurface` (`auth.ts`) check it. A FUTURE middleware
that gates `/api/v1` without checking the flag will silently re-block allowlisted public routes —
the "market-data route mysteriously 401s" bug, reborn.

1. At the flag-check in `requireAuth` and in `requireSurface` (`auth.ts`), add a comment, verbatim
   intent:
   ```
   // INVARIANT: any middleware that gates /api/v1 MUST check res.locals.bypassAuth first.
   // The public-route allowlist floor (index.replit.ts, conditionalApiV1Auth) sets this flag
   // for allowlisted paths (/auth, /ticker, /oppgrid, /webhooks/notarize, /clawdbot, token
   // shares, etc). Skipping this check silently 401s public routes. See AUTH_STEP4_DEBT.
   ```
2. At the floor definition (`index.replit.ts`, `conditionalApiV1Auth` / `API_V1_PUBLIC_PREFIXES`),
   add a comment pointing the other direction: this floor + the 15 legacy broad guards together
   enforce auth; the flag is what keeps the legacy guards from blocking the allowlist. Reference the
   debt doc.
   `file:line` for each comment.

## PART 2 — STEP-4 DEBT RECORD (make the deferral a decision, not amnesia)

Create `docs/audits/AUTH_STEP4_DEBT.md` stating, accurately:

- **Current architecture (the truth):** `/api/v1` auth is enforced by an INTENTIONAL floor
  (`conditionalApiV1Auth` + `API_V1_PUBLIC_PREFIXES`, 18 entries) PLUS ~15 LEGACY broad
  `app.use('/api/v1', requireAuth, ...)` guards left in place, all kept consistent by the
  `res.locals.bypassAuth` flag. Behaviorally correct; NOT the clean single-floor end state.
- **Why Step 4 was deferred:** converting the 15 broad guards to specific-path mounts
  (`app.use('/api/v1/<specific>', ...)`) requires editing the ROUTER FILES, because those routers
  define full-path routes (`router.get('/deals/:dealId/boundary')`) — remounting at a prefix would
  make Express strip the prefix and break every route. Invasive; the bypassAuth approach achieved the
  security goal without it.
- **What "done" (Step 4 complete) looks like:** the 15 legacy broad guards removed, routers remounted
  at specific paths (with their internal route paths adjusted), leaving ONE floor + allowlist as the
  single readable auth mechanism — no flag needed, no side-effect guards.
- **Residual risk while deferred:** the `bypassAuth` flag is a hidden invariant (Part 1 documents it);
  a new gating middleware that ignores it reintroduces the silent-401 bug. Acceptable, documented,
  revisit when router files are touched anyway.
- **Trigger to do it:** next time those router files are edited for another reason, or before scaling
  the team (more devs = higher chance someone adds a guard that ignores the flag).

List the ~15 legacy guard sites (`file:line` each) so Step 4, when done, has its exact target list.
