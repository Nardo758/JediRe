## 1. Purpose & Scope

The Tax Service computes the full tax picture for any deal across four sections that mirror the F9 Taxes tab:

- **Section A — Real Estate Tax** (annual ad valorem, caps, exemptions, **reassessment events across hold period**)
- **Section B — Tangible Personal Property (TPP)** (FF&E, appliances, equipment)
- **Section C — Income Tax & Depreciation** (federal depreciation, bonus dep, cost seg, state income tax, conformity)
- **Section D — Transfer Taxes & Doc Stamps** (deed transfer, intangible tax, county surtaxes, recording fees)

It is the single source of truth for every tax line item that flows into:

- F9 ProForma → Operating Expenses → Property Taxes (Section A + B; **year-by-year** with reassessment events)
- F9 ProForma → Below-Line → Depreciation Shield (Section C federal)
- F9 ProForma → Below-Line → Income Tax (Section C state)
- F9 Sources & Uses → Closing Costs → Transfer Taxes (Section D)
- M14 Risk → Tax Reassessment Risk (uses ruleset's trigger declarations)
- Deal Capsule → Cost Basis → Acquisition Costs (Section D)
- F9 ProForma → Annual Cash Flow → red REASSESSMENT EVENT bars (rendered from emitted events)

It is **NOT** an agent. It is a deterministic TypeScript service with pluggable rulesets per jurisdiction.

## 2. Architectural Principles

All inherited from existing platform docs; restated here so they apply unambiguously to this work.

1. **Deterministic over agentic.** Tax computation is `if X then Y` math. No LLM in the hot path. (`REPLIT_AGENT_IMPLEMENTATION_PROMPT.md` GR#8)
2. **Ruleset pattern from day one.** No `if (state === 'FL')` outside `rulesets/fl/`. (GR#10)
3. **Three-layer data model.** Rule mechanics in TypeScript, annual rates in versioned config, per-deal data via runtime fetch. Mixing layers is the bug.
4. **LayeredValue everywhere.** Every output field wraps in `LayeredValue<T>` with source provenance. Tier ordering: `t12_or_tax_bill` > `tax_service_computed` > `agent_research` > `unresolved`.
5. **Cache aggressively, invalidate by fiscal year.** Jurisdiction-level data (millage) cached per-county per-year; parcel data cached per-parcel per-year.
6. **Composite resolution.** Every property resolves three stacked rulesets: federal (always), state (one of FL/TX/GA/NC/...), county overlay (optional). The composer walks the stack.
7. **No UI changes required.** F9 layout (4 sections × 4-column grid + reassessment red-bar legend) already exists; this spec populates cells, doesn't restructure the tab.
8. **Triggers, not hardcoded years.** Reassessment events are emitted by ruleset trigger declarations, never hardcoded to "year 1" or "year of sale" outside the trigger evaluator.

## 3. Three-Layer Data Model

| Layer | Storage | Cadence | Owner | Example |
|---|---|---|---|---|
| **Rule mechanics** | TypeScript code in `src/services/tax/rulesets/` | Generational — rarely | Engineering | "FL caps non-homestead AV growth at 10%/yr"; "NC has cyclical reval, county sets cycle" |
| **Annual rate sheets** | Versioned config in `src/services/tax/rateSheets/` | Yearly — TRIM cycle, legislative session, IRC, county reval anchor years | Ops + Engineering | "Miami-Dade 2026 aggregate millage = 19.8344"; "NC-Mecklenburg cycle anchor = 2023, cycle = 4 yrs"; "2026 bonus dep = 20%" |
| **Per-deal data** | Runtime fetch via `PropertyAppraiserFetcher` + tax bill PDF parser | Per-deal | `LayeredValue` provenance | "464 Bishop assessed at $X; homestead = false; placed in service 2024-08-15" |

This split is the most important architectural decision in the spec. Violating it (e.g. embedding 2026 millage in `fl.ruleset.ts`, or hardcoding NC's 4-year cycle in `nc.ruleset.ts` instead of declaring it cyclical and putting cycle length in the county rate sheet) silently corrupts future deals and is the failure mode every refactor attempt has produced.

## 4. Composition Model

Every property has **three stacked rulesets**: federal, state, county. The Tax Service walks the stack per section.

```typescript
interface RulesetStack {
  federal: TaxRuleset;          // always present
  state:   TaxRuleset;          // FL, TX, GA, NC, ... or default
  county:  TaxRuleset | null;   // Miami-Dade, Harris, Fulton, Mecklenburg, Wake, ... optional
}

function resolveRulesetStack(deal: Deal): RulesetStack {
  return {
    federal: federalRuleset,
    state:   STATE_RULESETS[deal.state]                              ?? defaultStateRuleset,
    county:  COUNTY_RULESETS[`${deal.state}-${deal.county}`]         ?? null,
  };
}
```

### Section → Stack mapping

| Section | Federal | State | County |
|---|:-:|:-:|:-:|
| A · Real Estate Tax | — | ✅ (rules + triggers) | ✅ (millage, surtaxes, special districts, **cycle parameters**) |
| B · TPP | — | ✅ (does state tax TPP at all?) | ✅ (TPP millage, exemption threshold) |
| C · Income & Depreciation | ✅ (rates, bonus dep schedule, cost seg eligibility) | ✅ (state income tax rate, conformity) | — |
| D · Transfer Taxes | — | ✅ (deed stamp, intangible tax) | ✅ (Miami-Dade $0.45/$100 surtax) |

### Launch jurisdictions (v1)

- **Federal** — always
- **FL** — state + Miami-Dade, Broward, Palm Beach county overlays
- **GA** — state + Fulton county overlay
- **TX** — state + Harris county overlay
- **NC** — state + Mecklenburg, Wake county overlays
- **default** — fallback for unmapped states

NC is a launch jurisdiction specifically because it exercises the cyclical-trigger pattern that no other launch state exhibits. Without NC in the launch set, the trigger architecture is under-tested.

Not every level implements every method. State-only and county-only rulesets return `0` / `null` / `false` / `[]` from methods owned by other levels. The composer (§7) handles this cleanly.

### 5.1 Reassessment Triggers — Full Type Definitions

The trigger model is the v2 architectural addition. It replaces the v1 `reassessmentOnSale()` method.

```typescript
export type TriggerType =
  | 'sale'                       // qualifying ownership transfer
  | 'cyclical'                   // jurisdiction-mandated revaluation cycle
  | 'annual'                     // assessor's annual roll (most states)
  | 'new_construction'           // building permit / construction in progress
  | 'substantial_improvement'    // renovation exceeds threshold
  | 'use_change'                 // conversion (office → resi, etc.)
  | 'ownership_restructure'      // LLC reorg, partial transfer (out of scope v1)
  | 'appeal'                     // owner-initiated protest (downward only)
  | 'exemption_loss'             // homestead abandonment, abatement expiry
  | 'millage_change';            // rate change only; base unchanged

export type TriggerEffect =
  | 'full_reset'                 // assessed value → just/market value
  | 'sale_price_reset'           // assessed value → purchase price
  | 'cost_basis'                 // assessed value → land + (construction_cost × completion%)
  | 'co_reset'                   // full market reset at certificate of occupancy
  | 'partial_reset'              // only the improved portion is reassessed
  | 'phased_in'                  // change spread over N years
  | 'capped_reset'               // reset but capped at X% above prior
  | 'rate_only';                 // millage change; base unchanged

export interface ReassessmentTrigger {
  trigger_type: TriggerType;
  effect: TriggerEffect;

  // Cyclical specifics — populated for cyclical triggers
  cycle_years?: number;                        // e.g. NC Mecklenburg: 4; NC Wake: 8
  cycle_anchor_year?: number;                  // last revaluation year, e.g. 2023

  // New construction specifics
  cost_basis_during_build?: boolean;           // most jurisdictions: yes
  completion_pct_rule?: 'jan_1_snapshot' | 'continuous' | 'co_only';

  // Improvement specifics
  improvement_threshold_pct?: number;          // e.g. 25% of prior just value
  improvement_threshold_dollar?: number;

  // Phase-in specifics (for jurisdictions that phase large jumps)
  phase_in_years?: number;

  // Cap specifics (post-trigger annual cap, distinct from baseline annual cap)
  post_trigger_annual_cap?: number;

  notes: string;                               // human-readable description
}

export interface AssessmentCap {
  pct: number;                                 // e.g. 0.10 for FL 10%
  applies_to: 'all' | 'homestead_only' | 'non_homestead';
  baseline_rule: 'prior_year' | 'cap_inception_value' | 'last_qualifying_sale';
}

export interface DealTimeline {
  acquisition_year: number;                    // calendar year
  hold_years: number;
  exit_year: number;                           // calendar year

  // Development-specific
  placed_in_service_date?: Date;               // CO date; required for development variant
  construction_start_date?: Date;
  construction_milestones?: ConstructionMilestone[];

  // Redevelopment / value-add specific
  major_capex_events?: CapexEvent[];

  // Edge cases
  homestead_status_changes?: { year: number; status: boolean }[];
  abatement_expirations?: { year: number; program_name: string }[];
}

export interface CapexEvent {
  year_of_hold: number;                        // 1-indexed
  dollar_amount: number;
  scope: 'unit_renovation' | 'common_area' | 'mep_replacement' |
         'envelope' | 'addition' | 'gut_rehab';
}

export interface ConstructionMilestone {
  date: Date;
  completion_pct: number;                      // 0.0 to 1.0
  cumulative_cost: number;
}

export interface ReassessmentEvent {
  year_of_hold: number;                        // 1-indexed year of the deal
  calendar_year: number;
  trigger_type: TriggerType;
  effect: TriggerEffect;
  prior_assessed_value: number;
  new_assessed_value: number;
  delta_dollars: number;
  delta_pct: number;
  tax_impact_dollars: number;                  // delta × millage / 1000
  source_ruleset: string;                      // which ruleset emitted this event
  notes: string[];
}
```

### Refresh cadence

| Level | Cadence | Trigger |
|---|---|---|
| Federal | Annual (after Dec congressional sessions) + ad-hoc on bills passed | Engineering PR |
| State | Annual after legislative session ends | Engineering PR |
| County millage | Annual post-TRIM (FL: ~Sept 15; GA: typically June; TX: varies by CAD; NC: counties set during budget cycle) | Ops review + PR |

### Section A composition — year-by-year timeline walk

This is the v2 critical change. Section A is no longer a single annual computation; it walks the hold period and emits events.

```typescript
function computeSectionA(
  ctx: TaxContext,
  stack: RulesetStack,
  deal: Deal,
  timeline: DealTimeline
): SectionAResult {
  // 1. Collect all trigger declarations from the stack
  //    Federal contributes none. State + county both contribute.
  //    County-level cycle parameters override state-declared cycle.
  const declaredTriggers = mergeTriggers([
    ...stack.state.reassessmentTriggers(),
    ...(stack.county?.reassessmentTriggers() ?? []),
  ]);

  // 2. Evaluate triggers against the deal's timeline
  //    Each ruleset gets to emit events; events from county-overridden triggers
  //    take precedence over state defaults for the same trigger_type.
  const stateEvents = stack.state.reassessmentEventsForDeal(ctx, deal, timeline);
  const countyEvents = stack.county?.reassessmentEventsForDeal(ctx, deal, timeline) ?? [];
  const events = reconcileEvents(stateEvents, countyEvents);

  // 3. Build the year-by-year tax line
  const annualLines: AnnualTaxLine[] = [];
  let runningAssessedValue = ctx.assessed_value;

  for (let yearOfHold = 1; yearOfHold <= timeline.hold_years; yearOfHold++) {
    const calendarYear = timeline.acquisition_year + yearOfHold - 1;
    const eventThisYear = events.find(e => e.year_of_hold === yearOfHold);

    if (eventThisYear) {
      runningAssessedValue = eventThisYear.new_assessed_value;
    } else {
      // No event — apply jurisdiction's annual cap or natural growth
      const cap = stack.state.annualAssessmentCap();
      runningAssessedValue = applyAnnualGrowth(runningAssessedValue, ctx, cap);
    }

    const taxLine = stack.state.annualPropertyTax(
      { ...ctx, assessed_value: runningAssessedValue },
      calendarYear
    );

    annualLines.push({
      year_of_hold: yearOfHold,
      calendar_year: calendarYear,
      assessed_value: runningAssessedValue,
      tax_amount: taxLine.amount,
      reassessment_event: eventThisYear ?? null,
    });
  }

  return {
    annual_lines: annualLines,
    reassessment_events: events,
    millage_breakdown: stack.county?.millageBreakdown(ctx) ?? stack.state.millageBreakdown(ctx),
    exemptions: stack.state.realEstateExemptions(ctx),
    annual_cap: stack.state.annualAssessmentCap(),
    declared_triggers: declaredTriggers,
  };
}
```

### `reconcileEvents` — county overrides state for same trigger

```typescript
/**
 * When state and county both emit events for the same trigger_type in the same year,
 * county wins. This handles e.g. NC-Mecklenburg's 4-year cycle overriding NC's
 * default 8-year cycle declaration.
 */
function reconcileEvents(
  stateEvents: ReassessmentEvent[],
  countyEvents: ReassessmentEvent[]
): ReassessmentEvent[] {
  const merged = [...countyEvents];
  for (const stateEvent of stateEvents) {
    const overridden = countyEvents.some(
      ce => ce.year_of_hold === stateEvent.year_of_hold &&
            ce.trigger_type === stateEvent.trigger_type
    );
    if (!overridden) merged.push(stateEvent);
  }
  return merged.sort((a, b) => a.year_of_hold - b.year_of_hold);
}
```

**Implementation:** `GeneralPropertyAppraiserFetcher` routes internally:

1. **Tier 1** — tax bill PDF parser output (if user uploaded)
2. **Tier 2** — commercial aggregator (ATTOM primary, ~$0.05–0.50 per parcel)
3. **Tier 3** — county-direct adapter (only where built; optional optimization)
4. **Tier 4** — agent-flagged research (returns `null`; CashFlow Agent picks up via `web_search`)

County adapters are **internal implementation detail**, not first-class entries in any registry. Build only when ATTOM cost or staleness justifies it for a specific high-volume county.

## 11. The Agent's Narrow Role

The Tax Service is deterministic. Agents do not compute tax. They have exactly three roles, all out-of-band of the hot path:

### 11.1 New jurisdiction onboarding

Deal lands in a state/county with no ruleset (e.g. Charleston SC). Behavior:

1. `resolveRulesetStack` returns `state: defaultStateRuleset` and `county: null`
2. `taxService.forecast()` runs against defaults; returns results with `confidence: 'low'` and `notes: ['Using default ruleset; jurisdiction-specific rules not yet modeled']`
3. A Kafka event `jurisdiction_unmapped` fires; an Inngest function queues a Research Agent run
4. Research Agent uses `web_search` to locate state and county tax rules — including **declared trigger types and cycle parameters**, not just rates
5. Output written to `rate_sheet_versions` with `status='pending_review'`, `source='agent_research'`
6. Engineering reviews, refines, promotes to `status='active'`

### 11.2 Staleness flagging

Weekly cron reads all `active` rate sheets where `valid_through < NOW() + INTERVAL '30 days'`. For each:

1. Queues an agent task to re-verify the rates
2. Agent compares current `RateSheet` against fresh web research, surfaces deltas
3. If no changes detected, agent updates `valid_through` and increments minor version
4. If changes detected, agent drafts new sheet → `pending_review`

**Special case for cyclical-trigger jurisdictions:** When a county completes a revaluation, the cycle anchor year shifts. Agent flags this as a high-priority update. (Example: when Mecklenburg completes its 2027 revaluation, `cycle_anchor_year` shifts from 2023 to 2027, affecting every NC-Mecklenburg deal in the system.)

### 11.3 Edge cases the standard ruleset doesn't model

TIF districts, recently-passed abatement programs, special assessment liens, ownership_restructure scenarios. When `PropertyAppraiserFetcher` returns a parcel flag like `in_tif_district: true` and the ruleset has no TIF handler:

1. Tax line marked with `confidence: 'medium'` and `notes: ['TIF district detected but not modeled']`
2. Research Agent invoked once per parcel to characterize the program
3. Output stored in LayeredValue with `derived_from_search: true`, surfaces in F9 UI as a yellow advisory

In all three roles, agents **propose**, humans **approve**, code **executes**. The agent never writes to the runtime computation path.

## Appendix A — Section Composition Pseudocode

### A.1 Section A — Timeline Walk (canonical example)

```typescript
// FL Development deal acquired 2025, CO 2027, hold through 2032
//
// timeline = {
//   acquisition_year: 2025,
//   hold_years: 7,
//   exit_year: 2032,
//   construction_start_date: 2025-06-01,
//   placed_in_service_date: 2027-09-15,
// }
//
// FL ruleset declares two triggers:
//   1. { trigger_type: 'new_construction', effect: 'cost_basis',
//        cost_basis_during_build: true, completion_pct_rule: 'jan_1_snapshot' }
//   2. { trigger_type: 'sale', effect: 'sale_price_reset' }  // also fires year 1
//
// Trigger evaluator emits:
//   year 1 (2025): sale event — assessed_value = land + (construction_cost × completion%)
//                  (sale + construction merge into single event in development variant)
//   year 2 (2026): no new event — assessed value reflects more completion
//   year 3 (2027): co_reset event — assessed_value → full market value
//   years 4-7:     no events — annual cap (10% non-homestead) applies

function fl_reassessmentEventsForDeal(ctx, deal, timeline) {
  const events: ReassessmentEvent[] = [];

  if (deal.project_type === 'development') {
    // Cost-basis years
    for (let y = 1; y <= yearsUntilCO(timeline); y++) {
      events.push(buildCostBasisEvent(ctx, timeline, y));
    }
    // CO reset
    events.push(buildCOResetEvent(ctx, timeline));
  } else {
    // Existing or redevelopment → sale fires year 1
    events.push(buildSaleResetEvent(ctx, deal, year_of_hold: 1));
  }

  if (deal.project_type === 'redevelopment' && timeline.major_capex_events) {
    for (const capex of timeline.major_capex_events) {
      if (exceedsImprovementThreshold(capex, ctx)) {
        events.push(buildImprovementEvent(capex, ctx));
      }
    }
  }

  return events;
}
```

### A.2 NC — Cyclical Trigger with County Override

```typescript
// nc.ruleset.ts declares:
nc_reassessmentTriggers() {
  return [{
    trigger_type: 'cyclical',
    effect: 'full_reset',
    // cycle_years and cycle_anchor_year are NOT set at state level —
    // they MUST be provided by county rate sheet
    notes: 'NC mandates revaluation; counties set cycle length (4-8 years) and anchor.',
  }];
}

// nc-mecklenburg.ruleset.ts overrides via rate sheet:
//   trigger_parameters.cyclical = { cycle_years: 4, cycle_anchor_year: 2023 }
//
// nc-wake.ruleset.ts overrides via rate sheet:
//   trigger_parameters.cyclical = { cycle_years: 8, cycle_anchor_year: 2024 }

function nc_mecklenburg_reassessmentEventsForDeal(ctx, deal, timeline) {
  const sheet = loadRateSheet('nc-mecklenburg', timeline.acquisition_year);
  const { cycle_years, cycle_anchor_year } = sheet.trigger_parameters.cyclical;

  const events: ReassessmentEvent[] = [];
  for (let y = 1; y <= timeline.hold_years; y++) {
    const calendarYear = timeline.acquisition_year + y - 1;
    const yearsSinceAnchor = calendarYear - cycle_anchor_year;
    if (yearsSinceAnchor > 0 && yearsSinceAnchor % cycle_years === 0) {
      events.push({
        year_of_hold: y,
        calendar_year: calendarYear,
        trigger_type: 'cyclical',
        effect: 'full_reset',
        prior_assessed_value: ctx.assessed_value,
        new_assessed_value: estimateMarketValueAtYear(ctx, deal, y),
        // ...
        source_ruleset: 'nc-mecklenburg-2026.v1',
        notes: [`Mecklenburg County 4-year revaluation cycle (last: ${cycle_anchor_year})`],
      });
    }
  }
  return events;
}
```

### A.3 Section B / D — unchanged from v1

```typescript
function computeSectionB(ctx, stack, deal) {
  if (!stack.state.taxesTPP()) {
    return { taxed: false, annual_tax: 0, notes: [`${stack.state.jurisdiction} does not tax TPP`] };
  }
  const stateLine = stack.state.tppTax(toTPPContext(ctx), deal.year);
  const countyLine = stack.county?.tppTax(toTPPContext(ctx), deal.year) ?? null;
  return {
    taxed: true,
    annual_tax: mergeTppLines(stateLine, countyLine),
    exemption_amount: stack.state.tppExemptionAmount(),
    filing_requirement: stack.state.tppFilingRequirement(),
  };
}

function computeSectionD(ctx, stack, deal) {
  const stateBreakdown = stack.state.acquisitionTransferTax(deal.purchase_price, deal);
  const countySurtax = stack.county?.countySurtax(deal.purchase_price) ?? 0;
  const stateIntangible = stack.state.intangibleTax(deal.loan_amount);
  return {
    state_deed_stamp: stateBreakdown.state_deed_stamp,
    county_surtax: countySurtax,
    intangible_tax: stateIntangible,
    mortgage_doc_stamp: stateBreakdown.mortgage_doc_stamp,
    recording_fees: stack.state.recordingFees('deed', 4) + stack.state.recordingFees('mortgage', 12),
    total: /* sum of above */,
    formula: /* human-readable trace */,
  };
}
```
