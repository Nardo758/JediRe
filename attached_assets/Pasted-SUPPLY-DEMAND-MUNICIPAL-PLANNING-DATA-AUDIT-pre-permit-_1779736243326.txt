SUPPLY/DEMAND — MUNICIPAL PLANNING DATA AUDIT (pre-permit stage)

CONTEXT
The platform has assessor adapters for Atlanta-metro counties and 
Florida counties. Planning department data — pre-permit pipeline 
signals (zoning applications, site plan submissions, planning 
commission agendas) — is a separate data publisher and currently 
not integrated.

This dispatch audits what's publicly accessible for each metro 
where we have assessor adapters. No code changes. Output is a 
gap analysis informing future integration dispatches.

═══════════════════════════════════════════════════════════════════
PER-METRO INVESTIGATION
═══════════════════════════════════════════════════════════════════

For each metro:

A. ACCESS MECHANISM
   - Planning department portal URL
   - Platform type (ArcGIS Hub, Accela Civic Platform, Granicus, 
     proprietary, scraped HTML)
   - Is there a public API? Direct download?
   - Auth required? Rate limits?

B. STAGES OF PIPELINE CAPTURED
   For each metro, document which of these stages have public 
   data:
   - Zoning amendment applications (pending + recently approved)
   - Site plan / development plan submissions
   - Pre-application meetings (calendared)
   - Variance applications
   - Planning commission agendas (next 60 days)
   - Planning commission minutes (last 12 months)
   - Public hearing notices
   - Council agenda items mentioning development
   
   For each: structured (queryable) vs unstructured (PDF minutes 
   requiring extraction)

C. DATA FRESHNESS
   - How often is the source updated?
   - Is there a feed (RSS, webhook) or only polling?
   - Typical lag between filing and public visibility?

D. ENTITY LINKAGE
   - What identifiers does the planning data use?
   - Parcel number/ID (and does it match the assessor adapter's 
     parcel_id?)
   - Address?
   - Developer/applicant name?
   - Cross-reference key to data_library_assets / property_descriptions

E. RELEVANT FIELDS
   For each application/filing record, what fields are typically 
   available?
   - Project name
   - Applicant/developer name + contact
   - Property address + parcel
   - Proposed use (multifamily? mixed-use? commercial?)
   - Proposed unit count / sqft / stories
   - Current zoning + requested zoning (for amendments)
   - Submission date
   - Status (filed / under review / approved / denied / withdrawn)
   - Scheduled hearing date
   - Staff report / recommendation (often PDF)

F. EXTRACTION COMPLEXITY
   For each metro, rate:
   - GREEN: structured API with clean fields, direct query
   - YELLOW: portal with parseable HTML or downloadable CSV
   - ORANGE: PDF-based; requires OCR or LLM extraction
   - RED: no public access; would require FOIA or in-person requests
