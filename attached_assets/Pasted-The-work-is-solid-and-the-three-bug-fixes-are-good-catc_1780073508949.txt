On the divergence interpretation — "real and expected" deserves 
verification before treating as resolved. Three things worth

   vintage-match, submarket distribution. If Cobb County 
   class A comps are 60%+ of the set for a south Atlanta 
   class B subject, the 110bps spread is a comp quality 
   issue, not a methodology difference. Same check for 
   Atlanta MF #2.

2. Confirm whether the canary will include strategy-aware 
   comp selection (D-COMP-1 from the strategy-aware modules 
   document) or whether the canary is testing the 
   pre-strategy-aware version. 500 comps within 25 miles 
   isn't the intended end state; strategy-aware selection 
   narrows to 5-8 high-relevance comps. If canary tests 
   pre-narrowing, the canary is testing a worse version 
   than will ship — worth being explicit about which version 
   is being canaried.

3. Run the backtest harness against the new path's output 
   for the S1 deals (Atlanta MF #1, #2, Jacksonville). The 
   harness produces per-method per-deal price/cap error 
   against ground truth. If the new path's median price 
   error is worse than the legacy path's (pre-refactor 
   baseline), promotion to canary should pause regardless 
   of what the divergence summary shows.

The 110bps cap rate divergence, if it translates to 
proportional price divergence (~20%), would be larger than 
the legacy path's existing error. That would make the new 
path a regression, not an improvement. The backtest is the 
only test that distinguishes "different methodology, both 
valid" from "new methodology produces worse valuations."
