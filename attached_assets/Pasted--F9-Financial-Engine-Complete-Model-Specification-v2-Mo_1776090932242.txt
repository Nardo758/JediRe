## Part 1 — Architecture

### 1.1 The 12-tab structure (locked)

| # | Tab | Layer ownership | Computes | Read or Edit |
|---|---|---|---|---|
| 1 | **Overview** | All layers, AI on top | KPIs, S&U summary, AI synthesis | Read |
| 2 | **Pro Forma** | **Broker** | Year-0 as-is operating snapshot | Edit (corrections only) |
| 3 | **Assumptions** | **Platform → User** | Forward Y1-N drivers (rent, traffic, expenses, capex, exit) | Edit (full) |
| 4 | **Taxes** | **Platform → User** | Property tax model + income tax / depreciation + sale taxes | Edit (overrides) |
| 5 | **Sources & Uses** | **User-confirmed** | Closing costs itemization + capital stack origin | Edit (full) |
| 6 | **Debt** | **User-confirmed Platform-suggested** | Multi-loan stack: senior, mezz, pref, bridge, construction, refi | Edit (full) |
| 7 | **Capital & Waterfall** | **User-confirmed** | Equity tranches, pref, promote tiers, distribution rules | Edit (full) |

| 9 | **Returns** | **Engine** | Property + LP + GP returns, time-based metrics, hurdle deltas | Read |
| 10 | Sensitivity | Engine | Two-way heat maps, tornado | Toggle drivers |
| 11 | Decision | AI | Synthesis, risks, actions | Read |
| 12 | Compare | Engine + AI | Version diffs | Select versions |

### 1.2 The mental model

> **Pro Forma = Year 0 (what is)**
> **Assumptions + Taxes + Debt + Capital = Year 1-N rules (how it grows, what it costs, who pays, who collects)**
> **Projections + Returns = Output (what falls out)**

No tab computes anything that another tab also computes. Every number has exactly one home.

### 2.1 Pro Forma Tab (recap from v1, with valuation snapshot)

**Purpose:** Show the ingested as-is operating picture. Broker layer's home. Every cell is `LayeredValue<T>` with `broker` populated.

Three sections plus a top-of-tab valuation snapshot strip:

**Valuation Snapshot Strip (NEW — top of tab, sticky)**

A 6-tile strip showing the as-is metrics at the broker's asking price. Updates live as user corrects parser values:

| Tile | Value | Submarket Median | Percentile |
|---|---|---|---|
| Price / Unit | $X | $Y | nth |
| Price / SF (NRSF) | $X/SF | $Y/SF | nth |
| **GRM** | Xx | Yx | nth |
| **GIM** | Xx | Yx | nth |
| Going-In Cap (T-12) | X% | Y% | nth |

These sit at the top because they're the first sniff-test any underwriter applies. If GRM is 14x in a market where comps trade at 8-9x, the deal is dead before opening Assumptions.

Edits = corrections only (parser misreads). `Reset to ingested` per row, `Reparse from source` at top. Integrity banner flags when T-12 doesn't reconcile or rent roll doesn't tie to T-12 GPR within 3%.

### 2.3 Taxes Tab (NEW)

**Purpose:** Property tax is the single largest expense miss in broker proformas — Florida reassessment on sale resets the basis, often spiking taxes 40-80% in Y1. This tab models that explicitly. It also handles income tax / depreciation, sale-related transfer taxes, and TPP.

Florida-specific behaviors materially affect every deal in the primary market.

#### 2.3.1 Section A: Property Tax (RE Tax)

Per-year grid with the following rows. Each row LayeredValue (Broker / Platform / User).

| Row | Y0 (in-place) | Y1 (post-sale) | Y2 ... Yn |
|---|---|---|---|
| Just / Market Value | from county assessor | = Purchase Price (reassessment trigger) | + Platform appreciation model |
| Assessed Value | from county TRIM | = Just Value Y1 (reset) | min(prior × 1.10, Just Value) — **10% cap** |
| Taxable Value | Assessed − exemptions | same | same |
| Millage Rate | from TRIM | from TRIM (or platform-projected change) | Platform millage trend |
| **Gross RE Tax** | Taxable × Millage / 1000 | Taxable × Millage / 1000 | same |
| Early-pay discount | 4% if paid Nov | 4% Nov / 3% Dec / 2% Jan / 1% Feb | same |
| Special assessments (CDD, MSBU) | from county | from county | + escalation per CDD bond schedule |
| **Net RE Tax (Cash)** | T-12 actual | computed | computed |
| Tax Appeal scenario (toggle) | n/a | reduces taxable by X% | persists if won |

**The 10% non-homestead cap** is the structural mechanic Florida investors must model. Once a property is bought at a higher price, Year 1 reassessment hits the new Just Value. But going forward, annual assessed value increases are capped at 10% per year regardless of market value — until next sale or substantial improvement triggers reset. The grid shows this cap binding visually (lock icon when cap is the constraint).

**Reassessment event flag:** the Y1 row carries a banner "**REASSESSMENT EVENT** — Just Value resets to purchase price. Tax delta vs T-12: +$X (+Y%). This is the Tax Bump." Click for the math: `(Purchase Price × millage / 1000) − T-12 RE tax`.

**Platform value derivation:**
- Just Value Y1 = purchase price (from S&U tab)
- Just Value Y2-N = purchase price × (1 + market_appreciation_model_per_year)
- Assessed Value cap rule applied per year
- Millage Rate Platform value = trailing 5-year submarket millage trend (data: county GIS millage history)
- Special assessments pulled from county CDD database for Florida
- Tax appeal Platform recommendation: triggers when Assessed > 110% of comp-derived value

**Comparison row at top:** "T-12 RE Tax: $X | Year 1 modeled: $Y | Delta: +$Z (+W%)" — color-coded red if delta > +30%.

#### 2.3.2 Section B: Tangible Personal Property (TPP) Tax

FL only. Tax on FF&E of income-producing property. Material for furnished STR, hotel-style multifamily, or recently renovated assets.

| Row | Source |
|---|---|
| TPP Just Value (FF&E book value) | DR-405 filing or platform estimate from unit count × $/unit furnishing cost |
| TPP Assessed Value | = Just Value (no cap) |
| TPP Exemption ($25K) | automatic |
| TPP Millage | same as RE millage |
| **TPP Tax** | (Assessed − $25K) × Millage / 1000 |

Usually small ($5-15K/yr for typical multifamily) but missed in 70% of broker OMs.

#### 2.3.3 Section C: Income Tax / Depreciation Schedule

Per-year grid. This drives after-tax returns, which drives the LP-level return calculation.

| Row | Y1 | Y2 | ... | Sale Year |
|---|---|---|---|---|
| Acquisition basis | from S&U tab (purchase + closing + capex) | | | |
| Land value (non-depreciable) | platform: 15-25% of basis based on county | | | |
| Building value (depreciable) | Total basis − land | | | |
| 27.5-yr straight-line depreciation | Building / 27.5 | same | same | partial year |
| **Cost segregation election** (toggle) | accelerate 20-30% of basis to 5/7/15-yr | | | |
| Bonus depreciation rate (current year) | **2026: 40%, 2027: 20%, 2028: 0%** | per year schedule | | |
| Bonus depreciation $ | (segregated 5/7/15-yr property) × bonus rate | | | |
| **Total depreciation deduction** | sum | | | |
| NOI − Depreciation − Interest | taxable income | | | |
| Suspended losses carried fwd | track | | | |
| **Tax payable (before sale)** | applies user marginal rate | | | |
| Depreciation recapture at sale | min(depreciation taken, gain) × 25% | | | |
| Capital gains at sale | (sale − adjusted basis − recapture) × 20% | | | |
| **Total tax at sale** | recapture + cap gains | | | |
| 1031 exchange election (toggle) | defers entire tax bill | | | |

**Cost segregation Platform recommendation:** triggered when basis > $5M (study cost ~$8-15K is justified). Shows estimated NPV of accelerated depreciation vs study cost.

**Bonus depreciation timing alert:** since bonus depreciation phases to zero by 2028, the Platform flags closing-timing implications: "Closing in 2026 captures 40% bonus on $X segregated property = $Y immediate deduction. Closing in 2027 captures 20% = $Z. Delta: $W."

#### 2.3.4 Section D: Sale-Related Transfer Taxes

These hit at exit. Modeled in the sale year only.

**Florida Documentary Stamps on Deed:**
- Statewide: $0.70 per $100 of sale consideration = **0.7% of sale price**
- Miami-Dade: $0.60/$100 + $0.45/$100 surtax on non-single-family = **1.05% on multifamily**

**Florida Doc Stamps on Mortgage** (paid by buyer at refi or new debt origination):
- $0.35 per $100 of mortgage = **0.35% of loan amount**

**Florida Intangible Tax on Mortgage:**
- $0.002 per $1 of mortgage = **0.2% of loan amount** (one-time at origination)

**Combined FL closing tax burden on a new mortgage = 0.55% of loan amount, plus 0.7-1.05% of purchase price on the deed.** This rolls into Sources & Uses on the buy side and into sale proceeds calculation on the exit side.

#### 2.3.5 Tab-level outputs (read-only summary card top of tab)

- **Effective tax rate (RE tax / EGI)** Year 1 vs T-12 (delta highlighted)
- **Total tax expense (RE + TPP) Year 1 → Year N** sparkline
- **Tax bump impact on Year 1 NOI** ($, % of NOI)
- **Sale year tax burden** (recapture + cap gains + transfer) as % of gross proceeds
- **1031 deferral value** (NPV of taxes deferred)

### 2.4 Sources & Uses Tab (NEW)

**Purpose:** The capital stack origin and complete acquisition cost itemization. Total uses must equal total sources (enforced). This is the Year 0 financing snapshot — what hits the closing table.

#### 2.4.1 USES — Itemized

Grouped sections, every line user-editable with Platform suggested defaults:

**Acquisition**
- Purchase Price (from contract / OM, links to Pro Forma)
- Earnest Money credit (offset)

**Hard Diligence Costs**
- Appraisal — Platform default $8-15K based on size
- Phase I Environmental — $3-5K
- Property Condition Assessment (PCA) — $4-8K
- Survey (boundary + ALTA if required by lender) — $5-15K
- Zoning report — $2-5K
- Engineering / Structural review — varies
- Termite / WDO inspection — $500-1500
- Lead/asbestos screening (pre-1978 buildings) — $2-5K

**Soft Diligence Costs**
- Acquisition legal — Platform default 0.10-0.20% of price, min $40K
- Lender legal — 0.05-0.15% of loan, min $25K
- Travel & DD personnel — user input
- Background / OFAC — nominal

**Lender / Financing Costs**
- Loan origination fee — 0.5-1.0% of loan (toggle per lender type)
- Loan application fee — flat
- Lender title insurance premium — ALTA loan policy, ~0.3-0.4% of loan (FL Promulgated Rate)
- Lender legal (above)
- Mortgage broker fee (if used) — 0.5-1.0% of loan
- Rate cap purchase (if floater) — Platform pulls live rate cap pricing from Chatham/Pensford feed

**Title & Closing**
- Owner's title insurance — ~0.5-0.6% of price (FL Promulgated Rate)
- Endorsements (zoning, comp lien, survey, environmental) — bundled $2-10K
- Escrow / settlement fee — flat $2-5K
- Recording fees — nominal
- Closing agent fee — flat

**Florida-Specific Transfer Taxes** *(pulled from Taxes tab Section D)*
- Doc Stamps on Deed (0.7% statewide / 1.05% Miami-Dade)
- Doc Stamps on Mortgage (0.35% of loan)
- Intangible Tax on Mortgage (0.2% of loan)

**Required Reserves at Closing** *(lender-driven, pulls from Debt tab)*
- Tax & Insurance escrow (typically 3-6 months)
- Replacement Reserve initial deposit ($250-400/unit/yr)
- Immediate Repair Reserve (from PCA, usually 125% of identified)
- Interest reserve (bridge / construction loans only)
- Operating Deficit Reserve (for value-add)

**Sponsor Acquisition Fee**
- Acquisition fee — 1-2% of purchase price (paid to GP from equity)

**Working Capital**
- Initial operating reserve — typically 1-3 months OpEx

**Capex Funding** *(Year 0 portion of renovation budget)*
- Initial CapEx wave — links to Assumptions Section 4 (capex schedule)

**TOTAL USES**

#### 2.4.2 SOURCES — Itemized

- Senior Debt (links to Debt tab — pulls amount selected there)
- Mezzanine Debt (if applicable)
- Preferred Equity (if applicable)
- Sponsor Co-investment (GP equity)
- LP Equity Class A

**TOTAL SOURCES**

#### 2.4.3 Reconciliation

Top of tab: **`Sources $X — Uses $Y = $Z`** with green check if Z = 0 within $1, red banner otherwise. Equity gap (if uses exceed debt + GP) is auto-displayed as the "LP Equity Need." Capital tab consumes this.

#### 2.4.4 Closing Cost Benchmarks (right rail)

Platform shows a benchmarks panel:

- "Florida transfer taxes alone: $X (Y% of price). This is non-negotiable but often missed in pre-LOI underwriting."

### 2.5 Debt Tab (NEW detail)

**Purpose:** Multi-loan stack modeling. Handle every debt instrument used in commercial multifamily, plus refi events mid-hold. Populates Sources & Uses, drives debt service in Projections, drives DSCR and debt yield in Returns.

#### 2.5.1 Loan stack list

Top of tab: list of all loans on the deal. Multiple allowed. User adds/removes loans. Each row shows: Loan Type | Lender | Amount | Rate | Term | Status (proposed/locked/refi'd).

Statuses: `Proposed` (Platform suggestion), `Quoted` (user has actual term sheet), `Closed` (locked at acquisition), `Refi'd` (replaces prior loan in year X).

#### 2.5.2 Loan-level configuration

For each loan, a panel with these fields. All `LayeredValue` where Platform can suggest based on loan type + market.

**Identity**
- Loan Type — dropdown:
  - Senior Fixed (CMBS / Life Co)
  - Senior Floating (Bank / Debt Fund)
  - Agency — Fannie Mae DUS
  - Agency — Freddie Mac Optigo Conventional
  - Agency — Freddie Mac SBL (small balance)
  - Agency — HUD/FHA 223(f) or 221(d)(4)
  - Bridge (1-3 yr floating)
  - Construction Loan
  - Bridge-to-Perm
  - Mezzanine Debt
  - Preferred Equity (debt-like)
  - Subordinate Debt
  - Seller Carryback
  - Assumed Loan (in-place)
- Lender Name — free text
- Loan Position (1st mortgage / 2nd / mezz / pref)
- Origination Date
- Maturity Date

**Sizing**
- Loan Amount (or LTV / LTC trigger — Platform back-solves)
- LTV — auto-calc (loan / appraised value)
- LTC — auto-calc (loan / total cost from S&U)
- Debt Yield — auto-calc (Year 1 NOI / loan)
- DSCR (Year 1) — auto-calc
- Loan-to-Stabilized-Value (for value-add or development)

**Pricing**
- Rate Type — Fixed / Floating / Floating-with-cap
- Index (if floating) — SOFR (1mo, 3mo) / Prime / Treasury / WSJ Prime — pulls live from FRED
- Spread (bps over index)
- Floor rate
- Coupon (if fixed)
- All-in rate at close
- Note rate vs Pay rate (for IO portion)

**Term**
- Loan Term (years)
- Amortization Period (years) — separate from term
- IO Period (months)
- Balloon at maturity — auto-calc
- Extension Options — list: number, length, fee bps, conditions (DSCR test, debt yield test, fee paid)

**Fees**
- Origination Fee (bps of loan)
- Exit Fee (bps of loan, paid at payoff)
- Standby/Unused Fee (for construction undrawn balances)
- Servicing Fee (bps annually)
- Rate Cap Cost (for floaters) — Platform pulls from rate cap broker feed

**Prepayment**
- Lockout Period (months)
- Yield Maintenance / Defeasance / Step-Down / Open
- Step-Down Schedule (if step-down): year/% (e.g., 5,4,3,2,1,0)
- Open Period (last X months prepayable at par)

**Covenants**
- Min DSCR (test frequency — annual, quarterly)
- Min Debt Yield
- Min Occupancy
- Max LTV (mid-loan revaluation trigger)
- Cash trap thresholds
- Sweep thresholds

**Reserves Required by Lender**
- Tax & Insurance escrow
- Replacement Reserve $/unit/yr
- Immediate Repair Reserve
- Interest Reserve (bridge/construction)
- Carry Reserve (for lease-up loans)

**Recourse**
- Non-recourse / Limited carve-out / Partial / Full
- Burn-off triggers (DSCR test, debt yield test)
- Bad-boy carve-outs

#### 2.5.3 Amortization Schedule (per loan)

Read-only output, monthly granularity, with annual rollup. Columns:
- Period
- Beginning Balance
- Index Rate (if floating, sourced from forward curve)
- Coupon Rate
- Interest Payment
- Principal Payment (zero in IO period)
- Total Debt Service
- Ending Balance
- DSCR (NOI for period / DS for period)
- Debt Yield (NOI annualized / balance)

For floaters, the rate column uses **Platform forward curve** (SOFR forward strip from FRED) per period. User can override with flat-rate scenario or custom curve.

For construction loans, draws schedule (S-curve or user-input) ties to capex schedule from Assumptions Section 4.

#### 2.5.4 Refinance Event Modeling

User can add a "Refi Event" at any year. Refi event:
- New loan replaces old loan in Month X of Year Y
- Calculates payoff (existing balance + prepay penalty + exit fee)
- New loan proceeds — payoff = Net Refi Proceeds (distributed to equity per Capital tab waterfall)
- Triggers FL doc stamps + intangible tax on new loan (flows to Taxes tab)
- Resets amortization schedule from Month X forward

Platform refi recommendation: when remaining loan term < 24 months, when DSCR > 2.0x and rates have dropped 100bps, when deal hits stabilization milestones — flag as suggested refi event.

#### 2.5.5 Multi-Loan Aggregation

Bottom of tab shows aggregated stack:
- Combined Loan Balance by year
- Combined Debt Service by year
- Combined LTV / LTC / Debt Yield
- Weighted Average Cost of Capital (debt only)
- Stack visualization (bar chart by tranche)

#### 2.5.6 Loan-Level Metrics Panel (per loan, NEW)

For each loan, a metrics panel runs alongside the amortization schedule. These are the operative numbers for sizing, sanity-checking, and arguing with the lender.

**Sizing metrics (at close)**
- LTV (loan / appraised value)
- LTC (loan / total cost from S&U)
- Loan-to-Stabilized-Value (loan / stabilized value)
- Debt Yield (Year 1 NOI / loan)
- Debt Yield Stabilized (Stab NOI / loan)
- DSCR Year 1 / Min Year / Avg
- Loan Constant (Annual DS / Loan) — the all-in yield to the lender, the cost-of-debt for the GP

**Leverage diagnostic**
- **Positive Leverage Flag** — Going-In Cap > Loan Constant. Green check / red X.
- Leverage Spread (bps) — (Going-In Cap − Loan Constant) × 10000
- Levered IRR Lift — (Levered LP IRR − Unlevered Property IRR) bps contribution
- Cash-on-Cash Spread — Year 1 CoC − Loan Constant

If the indicator is red (negative leverage), the panel surfaces a warning: "This loan is more expensive than the property yields. Total return depends on appreciation, not cash flow. Common in low-cap-rate / value-add bridges, but verify the appreciation thesis is durable."

**Stress metrics**
- Break-Even Occupancy — occupancy at which NOI = DS
- Break-Even Rent — rent level at which NOI = DS
- DSCR @ -10% NOI
- DSCR @ +200bps Rate (floaters)
- Cash Trap Distance (bps)
- Default Buffer (months of DS in reserves)

**Maturity / refi metrics**
- Maturity Year & Balloon Balance
- Projected Maturity LTV (balloon / projected value)
- Projected Maturity Debt Yield (NOI at maturity / balloon)
- **Refi-Out Probability** — Platform model: P(maturity_LTV < market_refi_max AND maturity_DY > market_refi_min) using forward rate curve and projected NOI
- Defeasance / YM Cost Today (live calc)
- Suggested Refi Window (Platform-flagged years)

**Covenant cushion (live)**
- DSCR cushion vs minimum (current vs Year-of-min)
- Debt Yield cushion vs minimum
- Occupancy cushion vs minimum
- Distance to cash trap trigger

When any cushion drops below 10%, the panel turns amber. Below 5%, red. This is what saves you from accidentally signing into a covenant you'll breach in Year 2.

### 2.6 Capital & Waterfall Tab (NEW detail)

**Purpose:** Define the equity stack and the rules by which cash flows distribute. This determines LP returns, GP returns, sponsor promote earnings, and whether/when promote tiers crystallize.

#### 2.6.1 Equity Stack — Tranches

List of equity tranches. Each tranche editable. Order matters for distribution waterfall.

For each tranche:
- Tranche Name (e.g., "LP Class A", "LP Class B", "GP Co-invest", "Pref Equity")
- Tranche Type — Common / Preferred / Pref Equity (debt-like) / GP Carry-only
- Contribution $ (auto-fills from S&U "LP Equity Need")
- % of Equity Stack
- Pari Passu vs Sequential to other tranches
- Pref Return Rate (if preferred)
- Pref Compounding (annual / quarterly / monthly)
- Pref Type — Cumulative / Non-cumulative
- Catch-up provision (if any)
- Promote tier participation (yes/no)

#### 2.6.2 Waterfall Configuration

Distribution structure type — toggle:
- **American Waterfall** (deal-by-deal, distributions test on this deal alone)
- **European Waterfall** (fund-level, no promote until all LP capital + pref returned across the fund)

Tier-by-tier configuration. Standard 4-tier example, fully editable:

| Tier | Trigger | LP Share | GP Share | Notes |
|---|---|---|---|---|
| 1. Return of Capital | Until LP receives 100% of contributed equity | 100% | 0% | |
| 2. Preferred Return | Until LP receives X% IRR (or Y% accrued pref) | 100% | 0% | Pref rate user-set |
| 3. GP Catch-up | Until GP receives Z% of profits-to-date | 0% / 50% / 100% | 100% / 50% / 0% | Catch-up % editable |
| 4. Promote | All distributions thereafter | 80% / 70% / 60% | 20% / 30% / 40% | Multiple sub-tiers possible (IRR-based) |

**IRR-based promote tiers** (most common for value-add):
- 80/20 LP/GP up to 12% LP IRR
- 70/30 LP/GP from 12% to 18% LP IRR
- 60/40 LP/GP above 18% LP IRR

Each tier configurable: hurdle rate (IRR or EM), LP/GP split above hurdle, lookback provisions.

#### 2.6.3 Distribution Schedule (per period)

Read-only output. Periods = annual default, quarterly toggle. Per period shows:
- Available Cash for Distribution (from Projections — CFADS net of reserves)
- Cumulative Distributions to LP at start of period
- LP Pref Accrued at start of period
- Distribution per Tier (T1 / T2 / T3 / T4)
- LP Distribution this period
- GP Distribution this period
- LP IRR-to-date
- GP Promote earned this period
- Tier Currently Active flag

#### 2.6.4 GP Compensation Layers

In addition to promote, GP earns fees. These reduce LP returns. Fees configurable:
- **Acquisition Fee** — 1-2% of purchase price, paid at close (already in S&U Uses)
- **Asset Management Fee** — 0.75-1.5% of equity OR % of EGI annually
- **Construction Management Fee** — 3-5% of capex, if value-add
- **Disposition Fee** — 0.5-1.0% of sale price
- **Refinancing Fee** — 0.25-0.50% of new loan
- **Loan Guarantor Fee** — only if recourse, ~0.25% of loan

Each fee shown as $ per year on the Distribution Schedule. Toggleable on/off.

#### 2.6.5 Capital Stack Visualization

Stacked bar chart showing the full stack at close: from bottom (senior debt) to top (GP equity). Width = $, color-coded by tranche type. Hover shows IRR target per tranche.

Side panel: Weighted Average Cost of Capital (WACC) calc — debt-weighted-cost + equity-weighted-cost (using LP IRR target as equity cost). Compared to going-in cap rate to show the **leverage spread** (the GP's edge).

#### 2.8.5 Debt Metrics (expanded)

Debt analytics live in three groups: **coverage**, **structural**, and **stress**. Every metric tracked per year + min/avg/max + status vs covenant.

**Coverage Ratios**
- DSCR (Year 1, Min, Avg, Stabilized) — NOI / Total Debt Service
- Debt Yield (Year 1, Min, Avg, At Refi) — NOI / Loan Balance
- Interest Coverage Ratio (ICR) — NOI / Interest only (matters for IO loans where DSCR understates pressure)
- Fixed Charge Coverage — NOI / (DS + ground rent + capital lease) — for ground-leased deals
- Cash Flow Coverage — CFAS / DS (post-reserves coverage, the truest test)
- Loan Constant — Annual DS / Loan Amount (the all-in cost of debt as a yield)

**Structural Metrics**
- LTV at Close — Loan / Appraised Value
- LTV at Stabilization — Loan / Stabilized Value (Year 3 NOI / Exit Cap)
- LTV at Maturity — Balloon / Projected Maturity Value
- LTC — Loan / Total Cost (from S&U)
- Loan-to-Stabilized-Value (LTSV) — for value-add deals
- Refi-Out Probability — Platform model: P(maturity_LTV < refi_max_LTV AND maturity_DY > refi_min_DY) given forward rate curve
- Maturity Risk Score — composite of LTV-at-maturity, DY-at-maturity, market liquidity, and time-to-maturity

**Leverage Position (the GP's edge)**
- **Positive vs Negative Leverage Indicator** — Going-In Cap Rate vs Loan Constant. Positive when cap rate > constant; negative when constant > cap rate (you're borrowing at a higher rate than the property yields, only justified by appreciation thesis)
- **Leverage Spread (bps)** — (Going-In Cap − Loan Constant) × 10000
- **Leverage IRR Lift** — Levered LP IRR − Unlevered Property IRR (the bps that debt is contributing)
- **Cash-on-Cash Spread** — Year 1 CoC − Loan Constant (the immediate cash benefit of leverage)

**Stress Tests**
- Break-Even Occupancy — occupancy at which NOI = DS + reserves (the floor)
- Break-Even Rent — rent level at which NOI = DS at current occupancy
- DSCR at -10% NOI — coverage if NOI drops 10%
- DSCR at +200bps Rate (floaters only) — coverage if SOFR rises 200bps
- Cash Trap Distance — bps cushion above the cash trap DSCR trigger
- Default Buffer — months of debt service covered by reserves + working capital

**Refi Economics**
- Refi Event Summary (year, payoff, prepay penalty, exit fee, new loan, net proceeds returned to equity)
- Maturity LTV vs Origination LTV
- Defeasance / YM Cost Today (live calc using current Treasury curve from FRED)
- Cost-to-Refi-Now ($ + bps impact on IRR)

**Lender-Specific**
- Covenant Cushion at min year (DSCR, DY, occ vs covenant minimums)
- Sweep / Cash Trap Trigger Year (if any modeled period breaches)
- Recourse Burn-off Date (when carve-out triggers clear)

#### 2.8.6 Valuation Metrics (NEW)

The "what is this thing worth, in every framing investors use" section. Three sub-blocks: per-unit/SF metrics, multiples, and replacement cost.

**Per-Unit and Per-SF Pricing**

| Metric | Formula | Going-In | Stabilized | At Exit | Submarket Comp |
|---|---|---|---|---|---|
| Price per Unit | Total Cost / Units | $X | $Y | $Z | submarket median |
| Price per SF (Net Rentable) | Total Cost / NRSF | $X/SF | — | $Z/SF | submarket median |
| Price per SF (Gross) | Total Cost / Gross Building SF | $X/GSF | — | — | submarket median |
| Land Price per Buildable Unit | Land Cost / (Acres × Density) | $X/buildable unit | — | — | submarket median |
| Land Price per FAR SF | Land Cost / (Acres × FAR × 43,560) | $X/FAR SF | — | — | submarket median |

Each row shows your deal's value alongside the submarket median, with percentile position.

**Income Multiples**

| Metric | Formula | Going-In | Stabilized | At Exit |
|---|---|---|---|---|
| Cap Rate | NOI / Value | X% | Y% | Z% |
| Reverse Cap Rate (NOI Multiple) | Value / NOI | Xx | Yx | Zx |
| **GRM (Gross Rent Multiplier)** | Price / Annual GPR | Xx | — | — |
| **GIM (Gross Income Multiplier)** | Price / Annual EGI | Xx | — | — |
| Net Income Multiplier | Price / NOI | Xx | — | — |
| Operating Expense Ratio | OpEx / EGI | X% | Y% | Z% |
| Cash-on-Cash Yield | BTCF / Equity | X% | Y% | Z% |
| Yield on Cost (untrended) | Stab NOI / Total Cost | — | X% | — |
| Yield on Cost (trended) | Stab NOI w/ rent growth / Total Cost | — | X% | — |
| Development Spread | YoC − Exit Cap | — | — | bps |

**Why GRM/GIM matter:** They're the fastest gut-check metric brokers and small-balance lenders use. A GRM of 7-9x is healthy multifamily; >12x suggests overpaying or sub-market rents needing capture; <6x flags either distress or a market with extreme cap rates. The Platform shows GRM next to the submarket benchmark to surface mispricing fast.

**Replacement Cost & Intrinsic Value**

| Metric | Formula | Value | Benchmark |
|---|---|---|---|
| Replacement Cost (RC) | Land + Hard + Soft + Contingency from Marshall & Swift / RSMeans, FL-adjusted | $X | per construction cost feed |
| **Price-to-Replacement-Cost Ratio** | Total Cost / RC | X% | <80% = buy-existing arbitrage; >100% = build cheaper |
| RC per Unit | RC / Units | $X | submarket build comps |
| Insurable Value | RC − land − below-grade | $X | for insurance basis |

**Price-to-Replacement-Cost is the single most important "should I build or buy" gate.** When it sits below ~80%, existing product is trading at a meaningful discount to building new — buy-and-hold thesis. When it crosses 100%, the rational move is to develop. This is the same ratio that drives the BTS vs. Acquisition decision in Strategy Arbitrage (M08), surfaced here for the financial-side audience.

**Versus-Submarket Position Matrix**

A small 2x2 grid showing your deal's position on Price/SF (X-axis) vs Cap Rate (Y-axis) plotted against the submarket comp set. Quadrants:
- Low $/SF + High Cap → "Value Buy" (the goal)
- Low $/SF + Low Cap → "Suspicious" (why is yield low? operational issues?)
- High $/SF + High Cap → "Distressed Trophy" (premium asset, broken operations)
- High $/SF + Low Cap → "Trophy / Premium pricing"

#### 2.8.7 Time-Based Metrics

- Hold Period (months)
- Lease-Up Period (if dev/redev)
- Equity Recovery Period (months)
- Peak Equity Date
- Breakeven Cash Flow Date
- Years of Pref Accrual Before Crystallization

#### 2.8.8 Risk-Adjusted (from M14 Risk module)

- Probability-Weighted IRR (Σ scenario_prob × scenario_IRR)
- IRR Standard Deviation (from Monte Carlo, 1000 runs)
- Downside IRR (10th percentile)
- Sharpe-style ratio: (LP IRR − risk-free rate) / IRR std dev
- Tail Risk: probability of LP IRR < pref rate

Every write triggers cross-tab recompute via Kafka events (e.g., a Debt tab loan amount change → S&U recompute → equity gap recompute → Capital tranche resize → Projections recompute → Returns recompute).

## Part 5 — Cross-Tab Event Bus

Every user edit triggers a typed event on Kafka topic `deal.financials.changed`:

```ts
{
  dealId: string;
  tab: 'proforma' | 'assumptions' | 'taxes' | 'sources_uses' | 'debt' | 'capital';
  path: string;          // dot-path to changed cell
  oldValue: any;
  newValue: any;
  layer: 'broker' | 'platform' | 'user';
  timestamp: string;
  userId: string;
}
```

Recompute orchestrator (`financials-orchestrator.service.ts`) subscribes and triggers downstream rebuilds in dependency order:
- proforma change → assumptions baseline refresh → taxes recompute → projections rebuild → returns rebuild
- assumptions change → projections rebuild → returns rebuild → sensitivity invalidate
- taxes change → projections rebuild → returns rebuild
- sources_uses change → equity gap recompute → capital tranche resize → returns rebuild
- debt change → S&U sources update → projections debt service refresh → capital cfads refresh → returns rebuild
- capital change → distribution schedule refresh → returns rebuild

Recomputes run server-side, push results via WebSocket subscription so the user sees downstream tabs update without manual refresh.
