QUERY 1 — T12 TRASH FORMAT FREQUENCY (OQ-1)
═══════════════════════════════════════════════════════════════════

A. Identify T12 documents in data_library_files where 
   extraction_data contains utility line items
B. For each, classify trash format:
   - DISTINCT: has separate "Trash" / "Waste Removal" / "Garbage" 
     / "Refuse" line item
   - CONSOLIDATED: trash is part of a combined utilities line
   - ABSENT: no trash-related line item appears at all
C. Report:
   - Total T12s with utility line items: N
   - Distinct format count + percentage
   - Consolidated format count + percentage
   - Absent format count + percentage
   - Breakdown by market (Tampa, Orlando, Miami, Jacksonville, 
     Atlanta, Dallas)
D. Identify any pattern variations by operator/property type/vintage

QUERY 2 — UTILITY SPLIT RATIO CALIBRATION (OQ-3)
═══════════════════════════════════════════════════════════════════

A. Identify deals where BOTH conditions hold:
   - OM upload exists with combined utilitiesAnnual figure
   - T12 ingestion has itemized utilities (water, electric, gas, 
     trash as distinct lines)
B. For each matched deal, compute implied ratios from T12 itemization
C. Report per market:
   - Number of deals with both data sources: N
   - Per-utility implied ratios (median + range)
   - Standard deviation across deals (how consistent are the 
     ratios?)
D. If <5 deals per market: report market as INSUFFICIENT_DATA 
   and recommend deferring auto-split for that market
E. If ≥5 deals per market: report empirical ratios as the 
   calibrated defaults

QUERY 3 — LINE_ITEM_BENCHMARKS POPULATION (OQ-5)

   - Check refreshLineItemBenchmarks aggregator status
   - Identify whether the aggregator is running, when it last 
     ran, what's preventing population
   - Report findings (separate bug investigation may be needed)
