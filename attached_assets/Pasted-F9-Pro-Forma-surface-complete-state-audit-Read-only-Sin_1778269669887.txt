PHASE 0 — Locate the surface

The Pro Forma view is rendered by ProFormaSummaryTab.tsx (line range
known from prior audits). Confirm the path. Identify any sibling
components that contribute to the visible Pro Forma surface
(SourceBadge, F9SummaryBar, evidenceFilter logic, etc.).

INVENTORY — for every visible cell

For each row in the rendered Pro Forma, fill this table:

| Section | Row Label | Field Key | Layer | Source Status | Render Status | Notes |

Layer = which LayeredValue layer is being shown (broker / t12 /
rent_roll / tax_bill / om_narrative / platform / override / computed
/ derived).

Source Status (per layer that should populate):
  - WIRED_AND_POPULATED — read path works, cell has a value
  - WIRED_AND_NULL — read path works, source returned null (legitimate
    empty state)
  - WIRED_BUT_BLANK — read path works, source has a value, cell shows
    blank (rendering bug)
  - NOT_WIRED — no read path exists at all
  - PARTIALLY_WIRED — works for some deals/conditions but not others
  - DUPLICATE_SOURCE — multiple write paths to same field

Render Status:
  - RENDERS_CORRECTLY — value displays accurately
  - RENDERS_PLACEHOLDER — shows '--', 'Not Provided', 'Pending'
  - RENDERS_INCORRECTLY — shows a value but it's wrong
  - DOESN'T_RENDER — row hidden when it should be visible
  - SHOULD_NOT_RENDER — row showing when it shouldn't

For each row, audit ALL THREE primary layers (broker / platform /
override) plus any specialized layers (t12, rent_roll, tax_bill,
om_narrative, unit_mix). Note which are wired, which are dark,
which have data flowing.

DELIVERABLE STRUCTURE

The audit doc should produce three sections:

Section 1: Inventory table (the full row-by-row classification)

Section 2: Identified gaps grouped by category:
  - WIRED_BUT_BLANK gaps (rendering bugs — fix on the read/display
    side)
  - NOT_WIRED gaps (write path missing — fix on the source side)
  - PARTIALLY_WIRED gaps (works conditionally — needs scope investigation)
  - DUPLICATE_SOURCE gaps (multiple write paths to same field —
    same-shape problem as Purchase Price dual-source)

Section 3: Recommended priorities by impact:
  - P0: gaps where wrong number is silently shown to operator
  - P1: gaps where missing number prevents IRR/EM/CoC computation
  - P2: gaps where missing number degrades narrative/decision support
  - P3: cosmetic gaps (wrong placeholder text, missing badge, etc.)
