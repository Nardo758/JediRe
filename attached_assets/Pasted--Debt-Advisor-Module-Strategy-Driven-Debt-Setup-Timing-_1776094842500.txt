# Debt Advisor Module — Strategy-Driven Debt Setup & Timing

**Module:** M11 Debt (expanded) + new Debt Advisor layer
**Status:** Implementation-grade spec — ready for Claude Code
**Supersedes:** The Debt tab section of the F9 Complete Model Spec — adds the Advisor layer on top; preserves the existing modeling interface
**Dependencies:** M08 Strategies (v2 detection + plan output), `capital-structure.service.ts` (742 lines), `rate-sheet-parser.service.ts`, `rate-index.service.ts`, FRED forward rate curve feed, rate cap pricing feed (Chatham/Pensford), S2 Ownership Intelligence (lender activity)

---

## 1. Core Architectural Shift

Today's F9 Debt tab is a configuration surface — the user picks a loan type, fills in rate/term/fees, and the engine amortizes it. That's fine as a modeling tool, but it assumes the user already knows what product they should use. Most users don't — especially across asset classes and deal types.

The fix: **Strategy drives Debt.** Once M08 detects a sub-strategy and produces a plan, the Debt Advisor reads that output and recommends the optimal debt stack, sizing, timing, and lender targets. The user sees a recommendation first, the configuration interface second.

```
M08 Strategy Output                    Market Context
  • Detected sub-strategy                • FRED rates (SOFR curve, 10yr)
  • Plan phases + timing                 • Lender rate sheets
  • Value creation capex schedule        • Rate cap pricing
  • Exit target                          • Correlation signals (COR-08 etc.)
  • Monitoring triggers                  • Rate Strategy Score (RSS)
         │                                       │
         └─────────────┬─────────────────────────┘
                       ▼
          ┌─────────────────────────────┐
          │   DEBT ADVISOR SERVICE      │
          │                              │
          │   1. Product selection      │
          │   2. Sizing optimization    │
          │   3. Timing sequence        │
          │   4. Lender targeting       │
          │   5. Monitoring triggers    │
          └──────────────┬──────────────┘
                         ▼
          ┌──────────────────────────────┐
          │  Recommended Debt Plan       │
          │   Primary stack + refi events│
          │   Rationale per loan         │
          │   Alternatives per condition │
          └──────────────┬───────────────┘
                         │ user accepts
                         ▼
          ┌──────────────────────────────┐
          │  F9 Debt Tab (Configure)     │
          │  Auto-populated from plan    │
          │  User modifies as needed     │
          └──────────────────────────────┘
```

The existing Debt tab modeling capability stays intact — it's now the **Configure** view, reached by accepting (or overriding) the Advisor's recommendation.

---

## 2. The Debt Tab Becomes Two Views

**Advisor** (default landing view) — Strategy-driven recommendation with rationale, timing sequence, lender targets, and monitoring triggers. Read-mostly; edits through "Modify Recommendation."

**Configure** (existing modeling) — Full F9 Debt tab as specced: multi-loan stack, per-loan panels, amortization schedules, refi events, loan-level metrics panel. This is the raw interface for users who want to model a specific structure by hand.

Toggle at top of tab: **[ Advisor ] [ Configure ]**. Default = Advisor after Strategy runs. If Strategy hasn't run (or user overrode detection), Advisor shows "Run Strategy first" CTA.

This is the core library — maps each M08 sub-strategy to its recommended debt product(s) and structure. Lives in a config table editable without code changes.

| MF Core-Plus | Agency with supplemental allowance | CMBS 10yr, portfolio bank | 10yr / 30yr amort / 2-5yr IO / 70-75% LTV / supplemental in Y3-5 |
| MF Value-Add | Bridge-to-perm (bridge 2-3yr, refi to agency at stab) | Debt fund fixed 3-5yr | Bridge: 3yr+1+1 ext / IO full / 70-75% LTC / SOFR+250-350 / 65-75% LTSV |

| Sub-Strategy | Primary Product | Alternatives | Structure |
|---|---|---|---|

| Sub-Strategy | Primary Product | Alternatives | Structure |
|---|---|---|---|

### 3.7 The mapping logic

The Advisor first looks up the detected sub-strategy in the matrix, then applies **context modifiers**:

- Deal size modifier (agency DUS small-balance vs large; CMBS requires >$5M typically; SBL for <$7.5M multifamily)
- Geography modifier (some lenders don't lend in certain states/metros)
- Sponsor experience modifier (first-time sponsors can't access certain products)
- Asset age modifier (pre-1980 multifamily tougher for agency)
- Sponsor balance sheet modifier (recourse requirements scale with liquidity)

Output: primary product + 1-2 ranked alternatives, each with a fit score.

## 4. Debt Plan Formulator — Phased & Timing-Aware

The Advisor produces a **debt plan**, not a single-loan recommendation. The plan has phases that mirror the Strategy plan.

### 4.1 Debt Plan Structure

```
Phase 1 — Acquisition (Month 0)
  Product: [primary]
  Amount: $X
  Terms: Y yr / Z% rate / W bps fees
  Rationale: [why this product for this phase]
  Lender targets: [specific active lenders]

Phase 2 — Stabilization / Value-Creation (Months 1-24)
  Product continues (or extends)
  Monitoring: [triggers that could change the plan]

Phase 3 — Refi Event (Month 18-24)
  Product: [new permanent loan]
  Trigger conditions: stabilization metrics met (occ > 92%, DSCR > 1.35)
  Rationale: [why refi now vs hold or extend]
  Net proceeds to equity: $X
  FL doc stamps / intangible: $Y (cross-ref Taxes tab)

Phase 4 — Hold through exit (Months 24-36)
  Product continues
  Monitoring: [market triggers that could force early refi/exit]

Phase 5 — Exit (Month 36)
  Payoff: balance + prepay penalty if within lockout
  Net proceeds: $X (cross-ref Capital tab waterfall)
```

Every phase:
- Is tied to a specific month range that comes from the Strategy plan
- Carries a rationale (why this product, not the alternative)
- Has monitoring triggers that could change the plan before the phase runs
- Cross-references the F9 Taxes tab for transfer taxes at origination + refi events

### 4.2 Common Debt Plan Templates

Templates per sub-strategy. Example patterns:

**MF Value-Add template:**
```
M0: Bridge-to-perm close (bridge piece, 3yr + 1 + 1)
M3-24: Draw interest reserve if built in; deploy reno capital
M18: First look at refi (if stabilization ahead of schedule)
M24: Refi to Fannie DUS (target product)
M36: Exit — prepay DUS (yield maintenance if within lockout; defeasance cost calc'd live from Treasury curve)
```

**SFR BRRRR template:**
```
M0: Acquisition via hard money (12mo) or cash close
M1-6: Rehab + lease
M6: Refi seasoning clears (6mo for most conventional refi)
M7-9: Refi to DSCR loan at 75% ARV → return capital to equity
Hold indefinitely OR sell per strategy plan
```

**MF BTS Ground-Up template:**
```
M0: Construction loan (36-42mo, IO, interest reserve)
M1-24: Construction draws per vertical schedule
M25-30: C/O + lease-up begins
M30-36: Stabilization
M36: Decision gate — refi to permanent (Fannie DUS) OR sell at stabilization (if strategy = BTS)
```

**Retail NNN Core template:**
```
M0: CMBS long-term fixed at acquisition
No refi events expected — match loan to lease tail
M(Term-24): If within 2yr of maturity, begin refi/extension planning
```

Templates aren't rigid — the Advisor adjusts amounts/timing/rates based on current market and deal specifics. The template just provides the phase structure.

## 5. Market Context Integration

The Advisor doesn't recommend in a vacuum — it reads current market conditions to pick the right **flavor** of the product and the right timing.

### 5.1 Rate Environment Input (from FRED)

Live feed from `rate-index.service.ts`:
- 10yr Treasury
- SOFR (1mo, 3mo)
- SOFR forward curve (full strip)
- Fed Funds target
- Fed Funds futures (implied path)

**Rate Environment Classification** (updates nightly):
- **Dropping** — Forward curve shows SOFR declining > 50bps over 12mo
- **Flat** — Forward curve within ±25bps
- **Rising** — Forward curve shows SOFR rising > 50bps over 12mo

**Classification → Advisor Decisions**

| Environment | Fixed vs Floating | Term Length | Rate Cap Advice |
|---|---|---|---|
| Dropping | Floating (or short fixed with open prepay) | Shorter fixed 3-5yr | Buy cheap cap; priced in optimism |
| Flat | Either — decide on other factors | Match hold | Standard cap |
| Rising | Fixed, lock long | Long fixed 10yr+ | Cap expensive; consider higher strike |

Example narrative generated by the Advisor:

> "Forward curve shows SOFR at 3.1% today falling to 2.4% by Q4 2026. For a 3-year Value-Add bridge, floating SOFR+275 beats comparable 3yr fixed at 6.5% by an expected 40bps over the hold. Recommend floating with a rate cap at 4.5% strike ($380K cost, 2yr). Alternative if you want rate certainty: 3yr fixed at 6.5%."

### 5.2 Rate Strategy Score (RSS) Input

From the existing `rateStrategyScore.service.ts` (spec'd in `rate_strategy_formula.md`). The RSS components that bear on debt:

| RSS Component | Debt Implication |
|---|---|
| Debt-Cost-vs-History < 60 (debt expensive) | Shorter terms; refi when rates drop |
| Debt-Cost-vs-History > 80 (debt cheap) | Lock long; this is the pricing you want to keep |
| Cap-Spread-to-Treasury > 250bps | Buyer-friendly pricing; debt supports higher leverage |
| Cap-Spread-to-Treasury < 150bps | Over-levered market; reduce LTV recommendation |
| Rate Trajectory dropping | Floating favored |
| Rate Trajectory rising | Fixed favored |

### 5.3 Correlation Signal Input

Specific COR signals affect debt recommendations:

| Correlation | Debt Implication |
|---|---|
| COR-08 permit velocity > +40% sustained | Shorten exit window → use shorter loan term or open prepay. Don't lock a 10yr if you need to exit in 18mo. |
| COR-04 wage-rent reversal firing | Peak approaching → avoid long fixed; match loan term to compressed hold |
| COR-01 traffic surge + rent growth leading | Underwriting will improve — plan refi event to capture lower cap rate |
| COR-20 digital-physical gap > 15pp | Market repricing in 6-12mo → if refi planned, time it for after reprice |
| COR-06 rate trajectory dropping | Confirms floating or short-fixed recommendation |

### 5.4 Credit Market Conditions

Separately from Treasury / SOFR, spreads and lender appetite move:
- Agency (Fannie/Freddie) caps and pricing parameters — updated weekly from agency rate sheets
- CMBS new-issue spreads — tracks Trepp or similar
- Bridge lender activity — platform signal from rate-sheet-parser feeding Rate Environment module
- Construction lender appetite — tracks bank vs debt-fund share

If agency caps are constrained (hit quarterly limits), Advisor flags: "Fannie DUS limited capacity through Q1; pricing may widen 10-15bps. CMBS alternative at similar all-in."

## 6. Lender Targeting Engine

For each recommended product, the Advisor surfaces **specific active lenders** in the deal's market and size range. Pulls from S2 Ownership Intelligence + `rate-sheet-parser.service.ts` + platform lender database.

### 6.1 Lender Database Fields

Per lender:
- Name
- Product types offered (agency DUS, CMBS conduit, bridge, construction, SBL, etc.)
- Geography served (states, MSAs)
- Loan size range (min/max)
- Typical pricing vs benchmark (bps over SOFR for bridge, bps over Treasury for fixed)
- Typical LTV/LTC range
- Recourse preference
- Sponsor experience requirements
- Recent activity (deals closed last 90/180 days in this market)

### 6.2 Targeting Output

For each Advisor-recommended product:

```
Bridge-to-Perm Bridge (Phase 1)
Target size: $28.5M at 65% LTC

Active lenders in Tampa MSA, $20-40M bridge:
  1. Acore Capital — 6 deals YTD, typical SOFR+290, 75% LTC max, non-recourse
  2. Square Mile Capital — 4 deals YTD, SOFR+325, 70% LTC, limited guaranty
  3. Bank OZK — 8 deals YTD (regional bank, aggressive), SOFR+275, partial recourse

Best fit given your sizing (65% LTC, non-recourse preferred): Acore
Alternative if you want lower spread: Bank OZK (requires partial recourse)

[Request Quote] — template DM
```

When user clicks "Request Quote", the platform can open an email template pre-filled with deal facts, target terms, and the lender's contact. (Full quote-management feature is out of this scope — phase 1 is just surfacing names.)

### 6.3 Lender Database Sourcing

- Agency DUS lender roster (Fannie/Freddie publish the list)
- Platform rate-sheet ingestion (users forward rate sheets → parser extracts lender + terms; builds market benchmark over time)
- Transaction records (via ACRIS, CREXi, RCA feeds if accessible) showing which lender financed which deal
- User-contributed lender feedback (post-deal)

MVP lender database = agency DUS lenders + top-50 CMBS originators + top-50 bridge lenders by volume. Geographic filter applied.

## 7. Debt Monitoring Triggers

The Advisor doesn't stop at close — it registers monitoring triggers that feed the Strategy module's Monitoring Dashboard. When triggers fire, the Advisor recommends action.

Examples:

**Refi-window triggers:**
- "At M18 (stabilization target from Strategy plan), if occupancy > 92% AND DSCR > 1.35, agency refi eligible. Monitor weekly."
- "If 10yr Treasury drops 75bps below current, agency rate improves enough to justify refi even with prepay penalty. Estimated rate-lock savings: $X."

**Extension-vs-refi triggers:**
- "Bridge extension fee is 50bps (at M36). If bridge replacement pricing is within 25bps of current all-in, refi instead of extend (saves future-exit prepay cost)."

**Cap-replacement triggers:**
- "Rate cap expires M24. At current SOFR curve, replacement cap at same strike costs $180K. Budget in capital reserves."

**Covenant-breach triggers:**
- "DSCR projected to hit 1.15 at Y3 (covenant minimum 1.20). If breaches, cash trap activates. Consider reducing distributions in Y3 to preserve covenant cushion."

**Strategy-change triggers:**
- "If M08 re-detects a sub-strategy pivot (e.g., zoning upzone passes → BTS pivot), debt plan must restructure. Current bridge has 18-month minimum hold; early payoff = YM of $X."

**Market-condition triggers:**
- "If COR-08 permit velocity breaks +60% sustained 6mo, exit window compresses. Recommend early refi to long fixed now to lock pricing before market deteriorates."

Each trigger carries:
- The condition (exact numeric threshold)
- The current value
- The monitoring frequency
- The recommended action if the trigger fires
- The severity (info / warning / critical)

Triggers register to the same alert system used by Strategy plan monitoring, so they appear in the user's unified alert feed.

## 9. Backend Data Contract

`GET /api/deals/:dealId/debt/advisor` returns:

```ts
interface DebtAdvisor {
  dealId: string;
  computedAt: string;

  // Source: Strategy module
  strategyInputs: {
    subStrategyKey: string;
    holdMonths: number;
    phases: Array<{ phase: number; monthRange: [number, number]; capitalRequired: number; }>;
    exitTargetQuarter: string;
    exitBuyerType: string;
    monitoringTriggers: string[];   // COR IDs that could change the plan
  };

  // Source: Market context
  marketContext: {
    rateEnvironment: 'dropping' | 'flat' | 'rising';
    tenYrTreasury: number;
    sofrCurrent: number;
    sofrForwardCurve: Array<{ monthsOut: number; rate: number; }>;
    rss: { overall: number; components: Record<string, number>; };
    activeCorrelations: Array<{ id: string; debtImplication: string; }>;
    pricingWindow: 'favorable' | 'neutral' | 'unfavorable';
    pricingWindowRationale: string;
  };

  // Core output
  recommendedStack: Array<{
    phase: number;
    phaseLabel: string;
    monthRange: [number, number];
    product: {
      key: string;              // e.g. 'fannie_dus', 'bridge_to_perm', 'hard_money'
      name: string;             // human-readable
      rationale: string;
    };
    sizing: {
      loanAmount: number;
      ltv: number;
      ltc: number;
      ltsv?: number;
      debtYield: number;
      dscr: number;
    };
    pricing: {
      rateType: 'fixed' | 'floating' | 'floating_capped';
      index?: string;
      spread?: number;
      coupon: number;
      allInRate: number;
      rateCapDetails?: { strike: number; term: number; cost: number; };
    };
    term: {
      termYears: number;
      amortYears: number;
      ioMonths: number;
      extensionOptions: Array<{ length: number; feeBps: number; condition: string; }>;
    };
    fees: Record<string, number>;
    prepay: { type: string; schedule?: number[]; };
    recourse: { type: string; burnoff?: string; };
    reserves: Record<string, number>;
    lenderTargets: Array<{
      name: string;
      recentActivity: string;
      fitScore: number;        // 0-1
      typicalSpread: number;
      typicalLTC: number;
      recourseStance: string;
      whyBestFit: string;
    }>;
    monitoringTriggers: Array<{
      id: string;
      condition: string;
      currentValue: string;
      threshold: string;
      severity: 'info' | 'warning' | 'critical';
      actionOnTrigger: string;
    }>;
  }>;

  // Alternatives
  alternatives: Array<{
    scenarioName: string;        // "If sponsor wants zero rate risk"
    changes: string;             // "3yr fixed at 6.5%"
    allInCostDelta: number;      // basis points vs primary
    irrImpact: number;           // percentage points
    tradeoffs: string;
  }>;

  // Summary
  summary: {
    primaryProductNarrative: string;  // AI Coordinator synthesized "why this stack"
    totalClosingCosts: number;
    blendedCostOfCapital: number;
    leverageSpread: number;           // going-in cap vs blended loan constant
    positiveLeverage: boolean;
  };
}
```

`POST /api/deals/:dealId/debt/advisor/accept` populates the F9 Debt tab with the recommended stack as `platform`-layer values. User overrides in Configure sit on top as `user` layer.

`POST /api/deals/:dealId/debt/advisor/recompute` re-runs the Advisor (triggered automatically when Strategy re-detects or market context shifts materially).

## 10. Implementation Sequence

Each item = one Claude Code session unless noted.

1. **Strategy → Debt product mapping table** — build the config table (Section 3) with all sub-strategy → product mappings. Editable Google Sheets or JSON; not hard-coded. **1 session.**

2. **Context modifiers service** — `debt-context-modifier.service.ts`. Takes deal facts + sponsor profile and filters/re-ranks products (size, geography, age, experience, balance sheet). **1 session.**

3. **Rate environment classifier** — new `rate-environment.service.ts` that reads FRED forward curve + SOFR futures and classifies dropping/flat/rising. Updates nightly. **1 session.**

4. **Debt plan formulator** — `debt-plan-formulator.service.ts`. Takes Strategy plan + product mapping + market context → emits phased debt plan with sizing, pricing, timing, rationale per phase. **2.5 sessions** (the largest piece).

5. **Lender database + targeting service** — MVP database populated with agency DUS roster + top bridge/CMBS lenders. `lender-targeting.service.ts` filters by geography/size/product and ranks fit. **2 sessions.**

6. **Debt monitoring trigger registration** — extends the Strategy Monitoring Dashboard trigger system with debt-specific triggers (refi windows, cap replacement, covenant breach, market-condition gates). **1 session.**

7. **Advisor API endpoints** — `GET /debt/advisor`, `POST /advisor/accept`, `POST /advisor/recompute`. Wire Kafka event so Strategy re-detect or material market shift auto-recomputes Advisor. **1 session.**

8. **Frontend — Advisor view** — recommendation header, timeline swim-lane, per-phase detail panels, market context right rail, alternatives panel, monitoring triggers. **3 sessions** — this is the largest frontend piece.

9. **Frontend — Advisor ↔ Configure toggle + divergence flagging** — toggle UI, accept flow populating Configure, divergence banner when user modifies Configure vs Advisor. **1 session.**

10. **AI Coordinator narrative for debt plan** — endpoint generating the "why this stack" narrative citing strategy, rate environment, correlation signals, lender rationale. Cached 24h. **1 session.**

11. **Lender quote request template** — email/DM template generation for outreach to targeted lenders, pre-filled with deal facts. **0.5 session.**

**Total: ~15-16 sessions** for complete Debt Advisor build.
