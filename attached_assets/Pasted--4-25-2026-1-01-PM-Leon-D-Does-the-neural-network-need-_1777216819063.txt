Memory Architecture

The One Real Gap: Semantic Memory

Right now knowledge-graph/search does text matching only. To do "find me deals similar to this one" or "what markets behaved like Atlanta in 2019?" you need embeddings.

Two options:

1. OpenAI ada-002 — $0.0001 per 1K tokens, best quality (need OPENAI_API_KEY in Replit)
2. Local model via Ollama — free, slightly lower quality, needs setup

Everything else — the network already remembers through the database. Every agent run, every deal, every market event, every web search result gets persisted. The KG is the brain, the DB is the long-term memory.

Want me to wire up embedding generation so semantic search works? It's the last major missing piece.
