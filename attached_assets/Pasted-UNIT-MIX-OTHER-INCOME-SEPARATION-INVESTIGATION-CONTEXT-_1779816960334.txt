UNIT MIX + OTHER INCOME SEPARATION — INVESTIGATION

CONTEXT
Per Leon's UI fixes: split the current combined Unit Mix / Other 
Income surface into two separate pages, available across all deal 
types. Both pages need:

1. Auto-population from document upload (rent roll, OM, T-12)
2. Manual build-from-scratch path (especially for development 
   where no rent roll exists)
3. Unit Mix page: add leasing metrics and trends from rent roll
4. Other Income page: integrate suggestions from Program tab + 
   adoption timeline for new income sources

Investigate current state and design the changes. No code changes.

STEP 4 — DEVELOPMENT MODEL CONSIDERATIONS
═══════════════════════════════════════════════════════════════════

A. Development deals have no rent roll, no T-12. The manual-build 
   path is the only path. Document:
   - What's required for a complete unit mix from scratch?
   - What defaults can the platform suggest (from market comps, 
     M03 Dev Capacity output, similar developments)?
   - How does the user override / accept / refine these?

B. Validation:
   - Unit count must match the dev capacity (M03 massing output)
   - Per-unit sqft must be consistent with sqft constraints
   - Rent assumptions must align with market positioning

═══════════════════════════════════════════════════════════════════
STEP 5 — ADOPTION TIMELINE PATTERN
═══════════════════════════════════════════════════════════════════

This is a new input pattern; deserves its own design consideration:

A. Schema:
   - source_name (e.g., "EV charging fees")
   - start_period (months or year from acquisition / completion)
   - ramp_duration (months to reach steady state)
   - steady_state_value (per-unit per-month, or other unit)
   - probability_adopted (confidence the program will actually 
     be implemented)

B. How it feeds projections:
   - Year 1 may have $0 if program starts later
   - Year 2-3 ramps to steady state
   - Year N+ at full value × probability_adopted

C. Could this pattern apply elsewhere?
   - Lease-up timing (already handled by LVE)
   - Capex burn (already in CapEx schedule)
   - Concession burn-off (already in growth rates)
   - Other? (potential for reuse)
