## PROMPT 6: JEDI Score Engine

> **Paste this into Replit Agent:**

```
Build the JEDI Score calculation engine. This combines 5 sub-scores into a composite score.

TABLE: jedi_scores (new migration)
- id: UUID primary key
- property_id: UUID FK to properties, not null
- scored_at: TIMESTAMPTZ default now()
- jedi_score: DECIMAL(5,2), not null
- confidence: DECIMAL(4,2)
- demand_score: DECIMAL(5,2)
- supply_score: DECIMAL(5,2)
- momentum_score: DECIMAL(5,2)
- position_score: DECIMAL(5,2)
- risk_score: DECIMAL(5,2)
- bts_score: DECIMAL(5,2)
- flip_score: DECIMAL(5,2)
- rental_score: DECIMAL(5,2)
- str_score: DECIMAL(5,2)
- arbitrage_flag: BOOLEAN default FALSE
- recommended_strategy: VARCHAR(30)
- trigger_event: VARCHAR(100)
- input_snapshot: JSONB

Index: jedi_scores(property_id, scored_at DESC)

Create a Python service class JEDIScoreEngine with these methods:

calculate_supply_score(submarket_id):
  - Query latest market_snapshot for the submarket
  - Formula: supply_pressure = pipeline_units / (existing_inventory * absorption_rate * 12)
  - Where pipeline_units = pipeline_permitted + pipeline_under_construction from market_snapshot
  - supply_score = max(0, 100 - (supply_pressure * 100))
  - Return supply_score

calculate_demand_score(submarket_id):
  - For MVP, base on population growth and employment
  - Query latest 2 market_snapshots to get YoY change
  - demand_score = min(100, (population_growth_pct * 1000) + (employment_growth_pct * 500))
  - Clamp between 0-100
  - Return demand_score

calculate_momentum_score(submarket_id):
  - Query latest market_snapshot
  - rent_growth_component = min(40, rent_growth_yoy * 1000) — max 40 points from rent growth
  - vacancy_component = min(30, max(0, (0.08 - vacancy_rate) * 1000)) — lower vacancy = higher score
  - absorption_component = min(30, absorption_units / 10) — more absorption = higher score
  - momentum_score = rent_growth_component + vacancy_component + absorption_component
  - Clamp between 0-100

calculate_position_score(property_id, submarket_id):
  - For MVP, use a simplified version
  - position_score = 50 (placeholder — will be enhanced with submarket ranking later)

calculate_risk_score(submarket_id):
  - supply_risk = calculate from supply_pressure_ratio: if > 1.5 then 80, if > 1.0 then 60, if > 0.5 then 40, else 20
  - demand_risk = 100 minus demand_score (inverted)
  - risk_composite = (supply_risk * 0.5) + (demand_risk * 0.5)
  - risk_score = max(0, 100 - risk_composite) — higher score = LOWER risk
  - Return risk_score

calculate_jedi_score(property_id):
  - Get property's submarket_id
  - Calculate all 5 sub-scores
  - jedi_score = (demand * 0.30) + (supply * 0.25) + (momentum * 0.20) + (position * 0.15) + (risk * 0.10)
  - Also calculate strategy scores using different weights:
    BTS weights: demand 0.30, supply 0.25, momentum 0.20, position 0.15, risk 0.10
    Flip weights: demand 0.15, supply 0.20, momentum 0.30, position 0.20, risk 0.15
    Rental weights: demand 0.30, supply 0.25, momentum 0.20, position 0.15, risk 0.10
    STR weights: demand 0.25, supply 0.20, momentum 0.25, position 0.20, risk 0.10
  - arbitrage_flag = TRUE if max(strategy_scores) - second_max > 15
  - recommended_strategy = the strategy with highest score
  - Save to jedi_scores table
  - Return the complete score record

API endpoints:
- POST /api/v1/properties/{property_id}/score — calculate and save JEDI score
- GET /api/v1/properties/{property_id}/scores — get score history
- GET /api/v1/properties/{property_id}/score/latest — get most recent score

Run scoring for all 3 seed properties after creation.
```
