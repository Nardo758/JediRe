# F4 Markets Tab — Commentary Enhancement Specification

## Document Purpose
This document provides Claude Code with a comprehensive audit of all F4 Markets sections, identifies where narrative commentary is missing, and specifies implementation requirements for AI-generated market intelligence.

---

## Executive Summary

The F4 Markets module is the platform's market intelligence hub with a 5-level drill hierarchy: F4 Landing → MSA Terminal (12 tabs) → Submarket Terminal (8 tabs) → Property Card → Property Terminal.

**Core Finding**: Every tab displays metric grids and charts without contextual narrative. Users see numbers but receive no interpretation of what those numbers mean for their investment thesis.

**Solution**: Add Claude-generated commentary panels that synthesize metrics into actionable insights. Commentary is generated per-MSA/submarket dynamically, cached for 24 hours aligned with DealContext caching.

## Commentary Generation Architecture

### Approach: Claude-Generated Dynamic Commentary

Commentary is generated per-entity (MSA or Submarket) by Claude, not template-driven. This produces more insightful, contextual analysis.

### Data Flow
```
MSA Metrics (from API/store)
       ↓
Commentary Generation Request
       ↓
Claude API (claude-sonnet-4-20250514)
       ↓
Structured JSON Response
       ↓
Cache (24-hour TTL, aligned with DealContext)
       ↓
Frontend Renders Commentary Panels
```

### Commentary Request Schema

```typescript
interface CommentaryRequest {
  entityType: 'msa' | 'submarket';
  entityId: string;
  entityName: string;
  metrics: {
    // Core signals
    jediScore: number;
    demandScore: number;
    supplyScore: number;
    momentumScore: number;
    devCapacityScore: number;
    trafficScore: number;
    
    // Key metrics
    population: number;
    populationGrowth: number;
    medianIncome: number;
    employmentGrowth: number;
    avgRent: number;
    rentGrowth: number;
    occupancy: number;
    pipelinePercent: number;
    capRate: number;
    absorptionMonths: number;
    
    // Comparative context
    peerAvgJedi?: number;
    nationalAvgRent?: number;
    cyclePhase?: 'Expansion' | 'Peak' | 'Contraction' | 'Trough' | 'Recovery';
  };
  sections: CommentarySection[];
}

type CommentarySection = 
  | 'market_narrative'      // 2-3 paragraphs on market position
  | 'investment_thesis'     // Bulleted signals with ✓/⚠
  | 'signal_commentary'     // Per health-bar explanation
  | 'risk_watch'            // Active concerns
  | 'opportunity_signals'   // Where to buy
  | 'peer_context'          // Relative positioning
  | 'supply_narrative'      // Pipeline timing analysis
  | 'capital_narrative'     // Transaction market analysis
  | 'economic_narrative';   // Macro positioning
```

### Commentary Response Schema

```typescript
interface CommentaryResponse {
  entityId: string;
  generatedAt: string;  // ISO timestamp
  expiresAt: string;    // 24h from generation
  
  marketNarrative?: {
    summary: string;      // 1 sentence hook
    body: string;         // 2-3 paragraphs
    cyclePosition: string;
  };
  
  investmentThesis?: {
    signals: Array<{
      type: 'positive' | 'caution' | 'negative';
      text: string;
    }>;
    recommendation: 'Buy' | 'Hold' | 'Avoid' | 'Selective';
  };
  
  signalCommentary?: {
    demand: string;
    supply: string;
    momentum: string;
    devCapacity: string;
    traffic: string;
  };
  
  riskWatch?: {
    risks: Array<{
      severity: 'high' | 'medium' | 'low';
      title: string;
      detail: string;
    }>;
  };
  
  opportunitySignals?: {
    opportunities: Array<{
      submarket?: string;
      strategy?: string;
      rationale: string;
    }>;
  };
  
  peerContext?: {
    summary: string;
    strengths: string[];
    weaknesses: string[];
  };
  
  supplyNarrative?: string;
  capitalNarrative?: string;
  economicNarrative?: string;
}
```

## Commentary Prompt Template

Use this system prompt for Claude when generating MSA commentary:

```
You are a real estate investment analyst generating market intelligence for a Bloomberg Terminal-style platform. Your commentary must be:

1. PRECISE: Use specific numbers from the provided metrics
2. ACTIONABLE: Every paragraph should inform an investment decision
3. PROFESSIONAL: Write like a CBRE or JLL market report
4. CONCISE: Maximum 3 sentences per paragraph

For the Investment Thesis section, use:
- ✓ for positive signals (score > 70 or favorable trends)
- ⚠ for caution signals (score 50-70 or mixed trends)
- ✗ for negative signals (score < 50 or adverse trends)

Always connect metrics to investment implications:
- High DEMAND + Low SUPPLY = bullish for rent growth
- High PIPELINE + Slowing ABSORPTION = caution on timing
- Strong TRAFFIC + Weak CONVERSION = operational opportunity

Format your response as valid JSON matching the CommentaryResponse schema.
```

## Strategy Arbitrage Engine Metrics (M08)

The Strategy Arbitrage Engine simultaneously analyzes 4 investment strategies for every deal. These metrics should be incorporated into MSA and Deal-level commentary.

### 4 Strategies Analyzed
| Strategy | Label | Key Indicators |
|----------|-------|----------------|
| `build_to_sell` | Build-to-Sell | Absorption > supply pipeline, price appreciation accelerating, entitled land available, construction cost stability |
| `flip` | Flip | Distressed inventory rising, DOM declining, below-market distressed deals, renovation cost certainty |
| `rental` | Rental (Long-term) | Rent growth > inflation, vacancy declining, cash flow positive at acquisition, low tenant default risk |
| `str` | Short-Term Rental | Tourism/business travel up, STR permits available, ADR growth positive, favorable regulatory environment |

### Strategy Signal Weights
Each strategy weights the 5 core signals differently:

| Strategy | Demand | Supply | Momentum | Position | Risk |
|----------|--------|--------|----------|----------|------|
| Build-to-Sell | 30% | 25% | 20% | 15% | 10% |
| Flip | 15% | 20% | 30% | 20% | 15% |
| Rental | 30% | 25% | 20% | 15% | 10% |
| STR | 25% | 20% | 25% | 20% | 10% |

### Arbitrage Detection
- **Arbitrage Flag**: Triggered when delta between top strategies > 15 points AND top score > 70
- **Arbitrage Delta**: `recommended_score - second_best_score`

### Envelope Adjustments (Development Path)
Strategy scores are modified based on selected development path:

| Path | BTS | Flip | Rental | STR |
|------|-----|------|--------|-----|
| by_right | +8 | +2 | +5 | +3 |
| overlay_bonus | +6 | +1 | +7 | +2 |
| variance | +3 | +2 | +4 | +1 |
| rezone | +10 | -2 | +5 | -5 |

Additional modifiers:
- **Timeline penalty**: `entitlement_months × 0.5` (max 10), weighted by strategy sensitivity
- **Scale bonus**: +5 if >200 units, +3 if >100 units, weighted by strategy affinity

### Deal Signals Extracted for Scoring
```typescript
{
  supply_pressure: number;      // Inverted supply balance (0-1)
  demand_growth: number;        // Normalized demand score (0-1)
  rent_momentum: number;        // Rent growth normalized (0-1)
  job_growth: number;           // Employment trend (0-1)
  cap_rate_spread: number;      // Opportunity score (0-1)
  irr_potential: number;        // IRR / 30 (0-1)
  risk_score: number;           // Composite risk (0-1)
  regulatory_risk: number;      // Regulatory component (0-1)
  market_volatility: number;    // Supply-derived volatility (0-1)
  project_type: string;         // 'existing' | 'development' | 'redevelopment'
}
```

## JEDI Score Composition

The JEDI Score (0-100) is the platform's master composite signal used across all levels (MSA, Submarket, Property).

### Component Weights
| Component | Weight | Source |
|-----------|--------|--------|
| Demand | 30% | D-01 through D-12 signals |
| Supply | 25% | S-01 through S-10 signals |
| Momentum | 20% | M-01 through M-10 signals |
| Position | 15% | P-01 through P-12 signals |
| Risk | 10% | R-01 through R-10 signals (inverted) |

### Score Interpretation
| Score Range | Classification | Commentary Guidance |
|-------------|----------------|---------------------|
| 80-100 | Strong Buy | Lead with opportunity signals, minimal caveats |
| 60-79 | Buy / Selective | Balanced thesis, highlight entry timing |
| 40-59 | Hold / Neutral | Emphasize specific conditions for entry |
| 20-39 | Avoid / Caution | Lead with risks, identify turnaround catalysts |
| 0-19 | Strong Avoid | Focus on risk factors, exit strategies |

## Quadrant Classification

Deals/properties are classified into 4 quadrants based on JEDI Score and Traffic signals:

| Quadrant | Criteria | Commentary Tone |
|----------|----------|-----------------|
| Hidden Gem | High JEDI + Low Traffic visibility | "Undervalued opportunity, catalysts ahead" |
| Validated Winner | High JEDI + High Traffic visibility | "Market consensus correct, premium warranted" |
| Hype Risk | Low JEDI + High Traffic visibility | "Overvalued, correction risk, find the short" |
| Dead Weight | Low JEDI + Low Traffic visibility | "Avoid, no catalysts, dead money" |

## Lifecycle Phases

Market/submarket cycle positioning:

| Phase | Indicators | Commentary Focus |
|-------|------------|------------------|
| Emergence | Early demand signals, low supply, accelerating traffic | "Early mover advantage, position now" |
| Acceleration | Strong demand, supply responding, momentum positive | "Peak opportunity window, execute quickly" |
| Maturation | Demand stable, supply catching up, momentum flattening | "Selective entry, focus on differentiated assets" |
| Contraction | Demand softening, oversupply, negative momentum | "Risk management, exit underperformers" |
