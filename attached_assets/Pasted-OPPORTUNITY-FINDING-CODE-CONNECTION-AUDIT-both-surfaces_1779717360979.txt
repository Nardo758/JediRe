STEP 4 — CLASSIFY CONNECTION STATE
═══════════════════════════════════════════════════════════════════

For each opportunity-finding component:

A. CONNECTED — code runs, inputs populate, outputs reach users
B. PARTIALLY CONNECTED — code runs, some inputs populate, output 
   degrades gracefully (e.g., scoring works but is unreliable 
   due to missing data)
C. BUILT BUT DISCONNECTED — code exists, can't run usefully 
   because inputs are empty (this is the predicted state given 
   recent patterns)
D. BUILT BUT NEVER INVOKED — code exists, but the trigger that 
   should invoke it is missing or broken
E. SPEC ONLY — referenced in docs but no actual code
