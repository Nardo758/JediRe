ATLANTA GIS URL PROBE — M02 ZONING DATA SOURCE FIX

CONTEXT
M02 substrate shipped. RegulatoryConstraints interface, adapter 
pattern, FIPS routing, and the City of Atlanta zoning code lookup 
table (R-1 through OCR) are all working. 6 Atlanta properties 
verified end-to-end via paired-read.

What's not working: ATLANTA_GIS_URLS in atlanta.ts return 404 and 
"wrong org ID" from Replit. The architecture degrades correctly 
(jurisdiction resolves, source_chain records the attempt, zone_code 
stays null), but no actual zoning data flows through.

This dispatch finds the current working endpoint(s) and updates 
the URL constants. No structural changes needed.

═══════════════════════════════════════════════════════════════════
STEP 1 — IDENTIFY THE CURRENT WORKING ENDPOINT
═══════════════════════════════════════════════════════════════════

A. Document the currently-configured URLs in atlanta.ts:
   - Paste the exact URL constants
   - Note the specific error each returns (404 / wrong org ID / 
     timeout / etc.)
   - This is the "before" state for the closing note

B. Search for the current City of Atlanta zoning data endpoint. 
   Try these in order:

   1. gis.atlantaga.gov — the official City Planning GIS portal
      - Look for an ArcGIS REST services directory at:
        gis.atlantaga.gov/arcgis/rest/services
      - List MapServer / FeatureServer entries
      - Identify zoning-related layer (likely named "Zoning", 
        "Zoning_Districts", or similar)
      - Test the layer's /query endpoint with a known Atlanta 
        address or coordinates

   2. atlantaga.gov/government/departments/city-planning/maps-and-gis
      - May link to current zoning data services
      - Check the page's network calls in DevTools to find the 
        actual API endpoint backing any embedded maps

   3. opendata.atlantaregional.com — ARC's open data hub
      - Search for "Atlanta zoning" or "City of Atlanta zoning"
      - Note: ARC is regional, may aggregate but not host the 
        canonical City of Atlanta zoning

   4. ArcGIS Online directly — search hub.arcgis.com or 
      arcgis.com for "City of Atlanta zoning"
      - The "wrong org ID" error suggests the previous URL pointed 
        at an ArcGIS Online org that moved; finding the current 
        org's hosted layer resolves this

C. For each endpoint candidate found, verify from Replit:
   - HTTP request from the Replit environment (the actual deployment 
     context that matters)
   - Expected response: 200 with valid ArcGIS REST metadata
   - Sample a /query call with a known Atlanta lat/lng to confirm 
     zoning data returns

D. Document each candidate tested:
   - URL
   - Response (status code, sample payload if 200)
   - Whether it returned zoning code data
   - Whether it can be queried by lat/lng (preferred) or parcel_id 
     or address

REPORT FINDINGS BEFORE UPDATING THE CONSTANTS. STOP GATE.
