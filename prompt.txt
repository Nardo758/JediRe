## Implementation

### 1. Tab navigation on DataLibrarySettings
Modify DataLibrarySettings.tsx to add Bloomberg-style tab navigation: ASSETS | FILES | INBOX | ROLLUPS
- Tab routing via URL query param: ?tab=assets (default), ?tab=files, ?tab=inbox, ?tab=rollups
- Active tab has orange highlight
- Bulk Upload & Cloud Sync section stays at top visible across all tabs

### 2. Assets tab (default)
Keep existing asset database rendering as-is. Add FILES column showing count of files per asset. Count is clickable -> navigates to Files tab filtered to that asset's parcel_id.

### 3. Files tab
Copy the file browser from ArchiveLibraryPage.tsx into a subcomponent within DataLibrarySettings.tsx. Preserve ALL functionality:
- File count header
- Search (filename or property)
- Type filter chips (All / OM / T-12 / Rent Roll / Tax Bill / Leasing / Other)
- Status filter chips (All / success / partial / failed / unparsed)
- Table columns (FILENAME / PROPERTY / TYPE / STATUS / SIZE / UPLOADED + download)
- Pagination
Add cross-linking: property name links to Assets tab, ?tab=files&parcel_id=X filter support.

### 4. Inbox tab
Copy intake job queue from ArchiveInboxPage.tsx into subcomponent. Preserve all functionality.

### 5. Rollups tab (STUB)
