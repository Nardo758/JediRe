## General Build Pattern

For each section:

1. **Review** - Read placeholder content and feature list
2. **Design** - Sketch component structure
3. **Mock Data** - Create sample data
4. **Build UI** - Create components and layout
5. **Wire API** - Connect to backend (or mock)
6. **Test** - Manual and automated tests
7. **Polish** - Animations, loading states, errors
8. **Document** - Update section with screenshots

## Tips

- Start with static UI, add interactivity later
- Use placeholder data initially
- Build components bottom-up (small ‚Üí large)
- Test each component in isolation
- Commit after each major feature
- Ask for feedback early and often
