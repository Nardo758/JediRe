## 🛠️ Technical Approach

### Scraper Architecture
1. **Playwright/Puppeteer** - Headless browser
2. **Rate Limiting** - 1 request per 2 seconds (polite)
3. **HTML Parser** - cheerio/jsdom
4. **Pattern Recognition** - Identify zoning tables
5. **Data Extraction** - Parse district parameters
6. **Validation** - Check completeness
7. **Database Storage** - Save to zoning_districts table

### Data Points to Extract
From each zoning chapter:
- [ ] Zoning district codes (e.g., "R-1", "C-2")
- [ ] District names/descriptions
- [ ] Permitted uses
- [ ] Conditional uses
- [ ] Max density (units/acre or DU/acre)
- [ ] Max FAR (if specified)
- [ ] Max height (feet + stories)
- [ ] Min lot size
- [ ] Setback requirements (front/side/rear)
- [ ] Parking requirements (spaces/unit)
- [ ] Special conditions

### Legal Compliance
- ✅ Respect robots.txt
- ✅ Rate limit requests (polite scraping)
- ✅ Add disclaimer: "For educational/research purposes"
- ✅ Cache results (don't re-scrape unnecessarily)
- ⚠️ Monitor for cease & desist
