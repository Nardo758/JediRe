## Interaction Flow

### Enable/Disable Flow (Subscribed Module)
1. User clicks checkbox
2. UI updates immediately (optimistic)
3. API call: `PATCH /api/v1/modules/:slug/toggle { enabled: true/false }`
4. If error, revert UI and show error message

### Purchase Flow (Non-Subscribed Module)
1. User clicks checkbox or [Add Module]
2. Purchase modal opens
3. User clicks [Add Module ($XX/mo)]
4. API call: `POST /api/v1/modules/:slug/purchase`
5. Redirect to Stripe checkout or billing page

### Upgrade Bundle Flow
1. User clicks [Upgrade Bundle] in modal
2. Redirect to `/settings/billing` or pricing page

## Color Scheme

- **Primary Background**: White (`bg-white`)
- **Border**: Gray-200 (`border-gray-200`)
- **Text Primary**: Gray-900 (`text-gray-900`)
- **Text Secondary**: Gray-600 (`text-gray-600`)
- **Accent**: Blue-600 (`bg-blue-600`, `text-blue-600`)
- **Success**: Green-600 (`text-green-600`, `bg-green-100`)
- **Status Badge FREE**: Green (`bg-green-100 text-green-600`)
- **Status Badge Included**: Green (`bg-green-100 text-green-600`)
- **Status Badge Available**: Blue (`bg-blue-100 text-blue-600`)
- **Status Badge Price**: Gray (`bg-gray-100 text-gray-700`)
- **Plan Banner**: Gradient (`from-blue-50 to-purple-50 border-blue-200`)

## Responsive Behavior

- **Desktop (>1024px)**: Max-width 7xl, centered
- **Tablet (768-1024px)**: Full width with padding
- **Mobile (<768px)**: Stack elements vertically, full width buttons

## Accessibility

- Semantic HTML (`<h1>`, `<h2>`, `<button>`)
- ARIA labels on checkboxes: `aria-label="Enable Module Name"`
- Focus states on all interactive elements
- Keyboard navigation support
- Color contrast meets WCAG AA standards

## Performance

- **Optimistic UI**: Instant feedback on toggle
- **Lazy Rendering**: Categories only render when expanded
- **Memoization**: Module cards memoized to prevent re-renders
- **API Batching**: Single API call loads all modules
