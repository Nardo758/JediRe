**Priority: CRITICAL** — Highest deal volume in FL

| Field | Value |
|-------|-------|
| **Server** | `gis.miami.gov` (also `gis.miamigov.com`) |

## Connector Pattern

All endpoints use standard ArcGIS REST API patterns. Your existing connector handles:

```
GET {baseUrl}/{layerId}/query?
  where=1=1
  &outFields=*
  &geometry={bbox}
  &geometryType=esriGeometryEnvelope
  &inSR=4326
  &spatialRel=esriSpatialRelIntersects
  &outSR=4326
  &f=geojson
  &resultRecordCount={maxRecords}
```

**Key variations to handle:**
- Some servers use `MaxRecordCount: 1000` vs `2000` — implement pagination with `resultOffset`
- Spatial references vary (2881, 4326, 3857) — always request `outSR=4326` for normalization
- FeatureServer vs MapServer — prefer FeatureServer when both exist (supports editing/querying)
- Some older servers may not support `geoJSON` format — fall back to `json` and convert
