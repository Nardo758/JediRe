### AI Rendering Integration - ControlNet + SDXL
**Problem:** Users wanted photorealistic renderings (not just gray box massings)  
**Solution:** Integrated Replicate API with ControlNet to transform massing → market-quality renders

**Technology:**
- **ControlNet:** Preserves building structure/geometry
- **Stable Diffusion XL:** Adds materials, lighting, context
- **API:** Replicate (pay-per-inference, ~$0.02-0.05/render)

**Features:**
- **5 architectural styles:**
  - 🏢 Modern Glass
  - 🧱 Brick Traditional
  - 🏙️ Mixed-Use Urban
  - 🏭 Industrial Loft
  - 🌆 Luxury High-Rise
- **Lighting control:** Golden Hour / Midday / Dusk / Night
- **Weather control:** Sunny / Overcast / Rainy
- **Geographic context:** "Atlanta urban", "Miami beachfront", etc.
- **Screenshot capture:** Automatic from Three.js canvas
- **Processing time:** 15-30 seconds
- **Results:** Downloadable PNG

**Implementation:**
- Backend service: `ai-rendering.service.ts`
- API routes: `/api/v1/ai/render`
- Frontend panel: `AIRenderingPanel.tsx`
- Toolbar integration: "🎨 AI Render" button
- Setup guide: `/home/leon/clawd/jedire-ai-rendering-setup.md`

**How it works:**
1. User generates massing (e.g., 300-unit podium+tower)
2. Click "🎨 AI Render"
3. Select style + lighting + weather
4. System captures canvas screenshot
5. Sends to ControlNet (preserves structure) → SDXL (adds details)
6. Returns photorealistic image in ~20 seconds

**Cost:** ~$0.03/render (100 renders = $3, 1000 renders = $30)

### Key Decisions
- **Removed drawing tool:** Too slow, users don't want to manually trace polygons
- **Template-based generation:** Faster, produces valid building configurations
- **ControlNet over img2img:** Preserves structure (critical for architectural accuracy)
- **Replicate vs. self-hosted:** Faster time-to-market, no GPU infrastructure needed
